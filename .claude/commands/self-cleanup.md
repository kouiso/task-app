# /self-cleanup - 自分が使ったリソースだけを片付ける

**自分（このAIセッション）が作業中に使用・作成したリソースだけを片付ける。**

## 目的

作業完了時や作業中断時に、自分が使ったリソースを責任を持ってクリーンアップする。他のAIセッションや他の作業者が使っているリソースには一切触らない。

## 実行時の振る舞い

1. **自分が使ったリソースの特定**
   - このセッション中に起動したサーバー・プロセス
   - このセッション中に作成したデバッグセッション（mcp-debugger経由）
   - このセッション中に作成した一時ファイル
   - このセッション中に開いたDB接続

2. **クリーンアップ対象の提示**
   - 片付け対象のリソース一覧を表示
   - 各リソースの種類と状態を表示

3. **ユーザー確認（必須）**
   - 「これらを片付けますけどええですか？」と確認
   - **ユーザーの許可を得てから実行**

4. **クリーンアップ実行**
   - 許可されたリソースのみ片付け
   - 結果を報告

## 対象リソース

| カテゴリ | 対象 | クリーンアップ方法 |
|---------|------|-------------------|
| デバッグセッション | mcp-debuggerで開始したデバッグ | デバッグセッション終了 |
| 起動サーバー | このセッションで起動したdev server等 | プロセス終了 |
| 一時ファイル | デバッグ用に作成したファイル | ファイル削除 |
| DB接続 | postgres mcp等で開いた接続 | 接続クローズ |
| バックグラウンドタスク | `isBackground=true`で起動したタスク | タスク終了 |

## 重要な原則

- **自分のゴミは自分で片付ける** → 作業完了時は必ず実行を検討
- **他人のリソースには触らない** → このセッション以外で作成されたリソースは対象外
- **判断に迷ったら聞く** → 自分が作成したか不明なリソースはユーザーに確認

## 心得

- **立つ鳥跡を濁さず** → 作業完了時はクリーンな状態で終わる
- **他のセッションを壊すな** → 他のAIが使っている可能性のあるリソースは絶対に触らない
- **責任感** → 自分が使ったものは自分で片付ける
