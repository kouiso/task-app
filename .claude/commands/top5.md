# /top5 - 週次TOP5レポート自動生成

マネージャー向けの週次報告「TOP5」を自動生成する。リポジトリのgit履歴・PR・Issue対応履歴を分析し、今週の活動からビジネスインパクトのあるTOP5を生成する。

## 背景

a2zでは自由な働き方ができるからこそ、アウトプットを重要視している。TOP5とは単なる業務報告ではなく、あなたが取り組んでいる重要な活動や市場での気づきの中から、最も価値ある5つを厳選したもの。NVIDIAが実践している手法を取り入れている。

## 絶対ルール

1. **読み手は非エンジニア（マネージャー）** — 技術用語は一切使わない。「何がどう良くなったか」をビジネス言葉で書く
2. **出力はマークダウン形式** — コードブロック（```）で囲まない。Slackにそのままコピペできる形式で出力する
3. **補足は1行** — 各項目の補足は1行のみ。冗長に書かない
4. **先週との重複禁止** — 先週と同じトピックを入れない

### 技術用語の変換（必須）

技術的な成果は「ユーザーやビジネスにとって何が変わるか」に変換する。

| NG（技術用語） | OK（ビジネス言葉） |
|---------------|-------------------|
| JSON-LD構造化データを追加 | Google検索結果に記事情報が表示されるよう改善 |
| LCP改善・priority prop追加 | ページの表示速度を高速化 |
| CSP・X-Frame-Options追加 | 外部攻撃への防御策を追加 |
| N+1クエリ・O(n²)改善 | サーバーの処理速度を改善 |
| Firebase Auth Emulator対応 | 開発環境を外部サービスに依存しない構成に |
| E2Eテスト追加 | 品質チェックの自動化を整備 |
| 型安全性改善・as const修正 | コードの品質を改善し不具合が入りにくい状態に |
| CI/CDワークフロー追加 | 開発運用の自動化を整備 |
| ISRキャッシュ修正 | ページの読み込みが毎回遅くなる問題を修正 |
| sizes属性を最適化 | スマホでの画像読み込みを効率化 |

## 実行手順

### Step 0: 事前確認（最重要）

1. **スコープ確認**: ユーザーに「どのリポジトリの成果を中心にするか」を確認する。過去TOP5の傾向があれば判断材料にする
2. **先週TOP5の確認**: Slack `#-report-top5` の先週投稿、またはユーザーに先週の内容を確認し、重複を避ける
3. **期間特定**: 月曜日〜日曜日の期間を計算

### Step 1: データ収集

Git履歴・PR・Issueを並行で取得する。

```bash
# 全リポジトリのコミット数を一括確認
find /home/kouiso/ghq -maxdepth 3 -name ".git" -type d | while read gitdir; do
  repo=$(dirname "$gitdir")
  count=$(cd "$repo" && git log --since="<monday>" --until="<sunday+1>" --oneline --all --no-merges | wc -l)
  if [ "$count" -gt 0 ]; then echo "$repo: $count commits"; fi
done
```

対象リポジトリごとに:
```bash
gh pr list --state=all --limit 100 --json number,title,author,createdAt,mergedAt,state
gh issue list --state=closed --limit 50 --json number,title,closedAt
```

### Step 2: TOP5変換

- 作業報告→ビジネスインパクト表現に変換
- 「完成させる」「構築する」「確保する」等の動詞で表現
- 重要度順にソート（5つに厳選）
- **技術用語が残っていないかセルフチェック**

### Step 3: 最終出力

ユーザー確認後、コピペ可能な形式で出力する。

## ビジネスインパクト変換ルール

**単なる作業報告ではなく、価値創出につながる内容を表現すること。**

| NG（作業報告） | OK（ビジネスインパクト） |
|---------------|------------------------|
| 「○○を実装した」 | 「○○を完成させ、△△の向上に貢献」 |
| 「バグを修正した」 | 「□□の問題を解決し、ユーザー体験を改善」 |
| 「PRをレビューした」 | 「コード品質を確保し、チームの開発速度を維持」 |
| 「調査した」 | 「○○を分析し、次のアクションを特定」 |
| 「会議に参加した」 | 「○○の方針を決定し、チームの方向性を確保」 |
| 「顧客アンケートを実施した」 | 「顧客アンケートから次期製品の重要機能を特定」 |

## 効果的なTOP5の特徴

- 「完成させる」「構築する」「確保する」といった**行動を示す言葉**で表現
- **「何を成し遂げようとしているのか」**が明確
- 数値・具体的な成果があれば含める（「15%向上」「2800万円の損失回避」等）
- 補足は1行で文脈・次のアクションを補足

## 出力フォーマット

**コードブロックで囲まず、マークダウンのままユーザーに提示する。** ユーザーがそのままSlackにコピペできる形式が正解。

@mentor
🚀 TOP5
• [成果1のタイトル - ビジネスインパクト表現]
　　- [補足: 1行で具体的な数値、背景、または次のアクション]
• [成果2のタイトル]
　　- [補足: 1行]
• [成果3のタイトル]
　　- [補足: 1行]
• [成果4のタイトル]
　　- [補足: 1行]
• [成果5のタイトル]
　　- [補足: 1行]

## 出力例

@mentor
🚀 TOP5
• Google検索でサイトが目立つように改善し、新規ユーザー獲得の土台を整備
　　- 検索結果に記事の著者名・公開日が表示されるよう対応。クリック率の向上が見込める
• スマホ表示の不具合6件を一括修正し、ユーザーが離脱しにくいサイトに
　　- PMから指摘があったアニメーション見切れ・ヘッダー透過・横スクロールなどを対応
• ページの表示速度を改善し、ユーザーが「遅い」と感じる前に表示される状態に
　　- メインビジュアルの読み込み順を最適化し、ファーストビューの表示を高速化
• 外部攻撃への防御策を追加し、サイトの安全性を強化
　　- 不正な埋め込みやスクリプト攻撃を防ぐ設定を導入
• 今週だけで32件の改善を本番反映・25件の課題をクローズし、V1.0.1をリリース
　　- 全ページの品質チェック自動化を追加し、今後の品質劣化を防止

## 注意事項

- 活動が少ない週は無理に5つ埋めず、実際の成果数に合わせる
- 毎週同じ内容のTOP5が続く場合は、変化や成長が停滞している可能性を示唆
- 投稿先: Slack `#-report-top5` チャンネル
- 締切: 毎週日曜日まで
