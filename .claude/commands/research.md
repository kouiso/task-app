# /research - 情報収集・調査

作業に必要な情報を収集し、理解を深化させる。指示があるまで実装は行わない。

## 実行手順

1. リポジトリ全体を調査（コード、設定、ドキュメント）
2. 関連する Git 履歴・PR・Issue を確認
3. WebSearch/Tavily で外部情報を収集
4. 分かりやすい形で結果をまとめる
5. **実装は指示があるまで行わない**

## 調査対象

### コードベース
- ソースコード内容（Read, Grep, Glob）
- ディレクトリ構造
- 設定ファイル（package.json, tsconfig.json 等）
- ドキュメント（README, doc/ 配下）

### Git 履歴
- コミット履歴（`git log`, `git show`）
- 差分（`git diff`）
- 過去の変更理由（コミットメッセージ）

### 外部情報
- 公式ドキュメント
- ライブラリ仕様
- ベストプラクティス
- エラーメッセージの意味

## フォーマット

```markdown
## 調査結果

### 概要
[調査対象と目的]

### 関連ファイル
- [ファイルパス]: [役割・内容]

### 外部情報
- [参照元]: [内容の要約]

### 結論・推奨
[調査結果に基づく推奨事項]
```
