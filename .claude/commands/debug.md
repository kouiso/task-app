# /debug - バグ根本原因分析

根本原因を特定する。5-7個の可能性を列挙し、1-2個に絞り込む。修正前にログで仮説をテストする。

## 実行手順

1. エラーの症状を整理
2. 5-7個の可能性のある原因を列挙
3. ログ・コード分析で1-2個に絞り込み
4. 仮説をテスト（ログ追加、実行確認）
5. 不明なエラーは WebSearch/Tavily で必ず調査
6. 根本原因を特定し、修正案を提示

## フォーマット

```markdown
## バグ分析

### 症状
[観測されたエラー・問題の挙動]

### 可能性のある原因
1. [原因1] - 可能性: 高/中/低
2. [原因2] - 可能性: 高/中/低
3. [原因3] - 可能性: 高/中/低
...

### 絞り込み結果
- **最有力**: [原因X] - 理由: [根拠]
- **次点**: [原因Y] - 理由: [根拠]

### 検証方法
- [どうやって確認するか]

### 修正案
- [具体的な修正内容]
```
