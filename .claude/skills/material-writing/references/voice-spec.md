# お手本仕様書（ボイス仕様・床/天井・客観ルーブリック）

このファイルは、30日カリキュラム教材（`material/30days-curriculum/dayXX_*.md`）を書く・書き直すときの
合格基準を、機械と人が同じ物差しで判定できるようにまとめたものである。SKILL.md の思考手順（6手順・
5つの不在・なぜを教える）を実行したうえで、最後にこの物差しで合否を確認する。

由来: 外部レビュー（阿部・杉田）で「AI独特の言い回し・翻訳文感・完成品配布・用語が噛み砕けていない」を
指摘され「このままなら購入見送る」と判定された。実際に売れた他教材（Python基礎/tkinter/Nomagram動画講座）を
読み比べて参考にし、下限（床）と、それを超えるべき点（天井）を言語化した
（定量的な測定記録は残していない。床/天井は読み比べから導いた定性的な基準である）。

## 床（最低ライン。売れた他教材が満たしている構造。全dayで必須）

- 冒頭に Agenda / 今日のゴール / 完成イメージを先に出す（読者が地図を持ってから手を動かせる）
- 前日からの接続を1文置く（「Day N-1 の続きから」）
- 必要な前提・環境を明示する
- 1ステップ1トピック。手順を飛ばさない（「同様に」「残りも」「以下略」の禁止）
- コードブロックは25行以下。`// filepath:` 付き
- 表で列挙する（散文でだらだら並べない）
- 新出の専門用語は初出箇所で噛み砕く（読み方・役割・例え）
- スクリーンショットで要所を見せる
- 文体（ですます体・AI典型構文の禁止・偏愛語）は SKILL.md 本体の規則に従う
  （一覧はここに再掲しない。正は SKILL.md と hana652 guides）

## 天井（床を超えるために必須。売れた他教材の弱点をここで突き放す）

売れた他教材は「写経至上でなぜが薄い」「サンプル配布依存」だった。ここを逆にする。

- **なぜこう動くか** を各コードブロックの直後に必ず書く（手順書化の否定）
- **メンタルモデルを渡す**: 手続きの前にそれが解く問題を示し、既知の構造に結びつける
- **コア機能は1から作らせる**: その日の学習対象（コア機能）は読者が自分で書く導線にする。
  周辺コード（他ルーター・共通UI等）は scaffold が提供コードとして配布する構造を採っており、
  教材の売り文句もこの実態（コア自作＋周辺は提供コード）に合わせて正直に書く。完成品を配って
  登録させるだけの日を作らない
- **つまずき対処** を用意する（その日のエラーと直し方）
- 淡々としすぎず、初心者への親しみを持つ（ただしAI臭・関西弁は入れない）

## 客観ルーブリック（これを全て満たしたら reviewer-grade = 合格）

| # | 判定項目 | 判定手段 |
|---|---------|---------|
| 1 | 地の文に常体混入ゼロ（コード・コメント・アプリ表示文字列は除外） | 目視 + check_tone.py + 常体全数チェック |
| 2 | 英語直訳調ゼロ（`することが可能です`/`これにより`/`〜と言えるでしょう`/`さまざまな` 等） | 目視 + grep |
| 3 | 各コードブロック直後に「なぜ」がある | 目視 |
| 4 | 新出専門用語に初出注釈（カリキュラム内で未注釈の用語は1個でもFAIL。前のdayで注釈済みの再出はWARNING） | check_quality.sh Step 8 |
| 5 | 比喩・例え話は直後に実体（コード・ファイル・動作）へ着地。比喩で締めない | 目視 |
| 6 | スクショ参照が一意（md5重複なし）かつキャプションと画像が整合 | md5sum + 目視 |
| 7 | Gate1 textlint 0 / Gate2 check_quality 全PASS | ツール実行 |
| 8 | Gate3（冷えた別エージェントの初心者シミュレーション）で詰まり3件以上を回収・修正 | サブエージェント |
| 9 | Gate3 の初心者シミュレーションが、その日の追加課題（類似手続き）を本文の情報だけで再現できた | Gate3（再現失敗＝FAIL、判定ログを残す） |

## 機械が見逃す層（ゲート通過≠品質。ここを人/Gate3で必ず補う）

- **check_tone.py は一部の常体混入を PASS する**（実証: day09:892「書けた」/ day01系の script 出力）。
  よって常体は目視 + 全数チェックで別途確認する。
- **textlint は語り口・翻訳文感（register）を対象にしない**（SKILL.md 記載、確信度75%）。
  AI臭・翻訳文感は Gate3 の冷えた読者で拾う。
- **スクショの重複・誤ラベル**は textlint/check_quality の対象外（実証: day09 cards.png==complete.png）。
  md5重複とキャプション整合を別途確認する。

## 運用（1日1サイクル）

1. 現物を全読
2. SKILL.md 6手順で書く前に語彙矯正
3. 書き直し（床維持・天井で超える）
4. Gate1/2 + 常体全数 + スクショ整合
5. Gate3（初心者シミュレーション）
6. 全緑まで直して1日1コミット

触ったファイルは毎回 Gate1/2 を単体で回し、加えてディレクトリ一括
（`bash scripts/curriculum-qa/check_quality.sh material/30days-curriculum/`。day/appendix が対象）でも
全緑を確認する。CI（material-gate.yml Gate2）も同じスコープで毎PR全件を回すので、宣言と実態が
ずれたらどちらかが壊れている合図である。

**表面指標だけで「完了」と報告しない。** reviewer-grade 到達 / first-pass どまり / 未着手を真状態で報告する。
