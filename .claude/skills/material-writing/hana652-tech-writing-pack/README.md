# Tech Writing Pack — 技術ブログのためのSEO・作文パック

プログラミング/技術ブログを書く人のための、**SEO** と **作文（AI臭を消す・自然な日本語にする）** のルール集。
「記事の書き方がよくわからない」「AIに下書きさせると、なんか機械っぽくなる」を解決するためのパッケージです。

このパックの特徴は、**コロケーション（語と語の自然な結びつき）の崩れ**を専用ガイドで手厚く扱っていること。
ChatGPT / Claude が生成した日本語でいちばん不自然になりやすいポイントを、検出と直し方まで含めてカバーしています。

> 📊 **まずはこれを見るのがおすすめ**：同じ記事の「AI臭版」と「人間版」を横並びで比較
> → **https://hana652.github.io/tech-writing-pack/examples/comparison.html**

## 読む順番（はじめての人向け）

最初から全部読まなくて大丈夫。下の順で、必要なところだけ。
（イメージを先につかみたい人は、まず **`examples/comparison.html`** をブラウザで開くのがおすすめ。同じ記事の「AI臭版」と「人間版」を横並びで見られます。）

1. **まずは `guides/01-seo-for-dev-blogs.md`** ── 「誰の何を解決する記事か」を決める。これがSEOの9割
2. **記事を書くとき `guides/02-article-structures.md`** ── 書く記事のタイプに合うテンプレを選ぶ
3. **AIに下書きさせたら `guides/03-anti-ai.md` と `guides/04-collocation.md`** ── ここで「AIっぽさ」を消す
4. **正確さが気になったら `guides/05-technical-writing.md`** ── コードの説明・手順の正確さはここ
5. **直すとき `guides/06-rewriting-prompts.md`** ── AIに直させるプロンプトをコピペで使う
6. **公開前に `checklist.md`** ── 1枚で最終チェック

迷ったら **「01 → 03 → 04 → checklist」** の最短ルートだけでも効果あり。

## このパックの使い方（3パターン）

### パターンA：自分で書く
`01`（設計）→ `02`（構成）で書く → `03`/`04`（AI臭・コロケーション）と `05`（正確さ）で見直す → `checklist`。

### パターンB：AIに下書きさせて整える（おすすめ）
1. `01` で記事の目的・読者・KWを決める
2. AIに下書きを書かせる（`06` のCARTEテンプレを使うと精度が上がる）
3. `06` の「まとめプロンプト」を別のAIに投げて一括で書き直させる
4. `scripts/ai-smell-lint.py` で機械チェック → 残った箇所を手で直す
5. `checklist` で最終確認

### パターンC：チェックだけ使う
すでに書いた記事を `scripts/ai-smell-lint.py` に通して、AI臭・コロケーション崩れの候補を洗い出す。

```bash
python3 scripts/ai-smell-lint.py path/to/article.md
```

## 機械チェックの使い方

Python 3 があれば追加インストール不要で動きます。

```bash
# 1ファイル
python3 scripts/ai-smell-lint.py article.md

# 複数ファイルまとめて
python3 scripts/ai-smell-lint.py posts/*.md
```

- 終了コード `0` ＝ 致命的な問題なし（warnは目視で要否判断）
- 終了コード `1` ＝ 致命症状あり（記号NG / 章境界アナウンス等）。直してから公開

スクリプトはあくまで**候補を出す道具**。最終判断は人がやってください。正当な文脈（コード内の `—` など）を誤検出することもあります。

---

## 大事な前提

このパックのルールは **「禁止リストを暗記する」ためのものではありません**。
AI文章が不自然になる**根本原因**（`03-anti-ai.md` の「5つの不在」）を理解すれば、リストに無い表現も自分で直せるようになります。

ルールは破ってもいい。**なぜそのルールがあるかを理解した上で、意図して破る**なら、それは「個性」になります。
