# 04. コロケーション崩れ・AI偏愛語大全

**このパックの目玉。** ChatGPTやClaudeが生成した日本語でいちばん不自然になるのが、ここで扱う「語と語の結びつきのズレ」です。

崩れには2系統ある。

1. **AI偏愛語の"擦り倒し"**（最頻発・最重要）── AIには「落ちる」「壊れる」「設計」「本質」のようなお気に入りの語があり、合わない文脈にまで被せる。「その動詞、そこで使う？」が大量発生する
2. **動詞・助詞・形容詞の選択ミス** ── 「課題を改善する」のように、近い意味の別の語で慣用句を壊す

世の中の「AI臭の消し方」記事はほとんど 2 までしか触れていない。けれど読んで不自然なのは圧倒的に 1。**しかも 1 は厄介で、「AI臭を批判する記事」の筆者ですら無自覚に使ってしまう**（後述）。だからこのガイドは 1 を最上位に置く。

---

## なぜAIは特定の語を擦り倒すのか

LLMの仕組みから説明できる。

- AIは「次に来る確率が最も高い語」を選ぶ。だから学習データで「それっぽく」高頻度だった比喩語（落ちる・刺さる・本質・設計…）に**収束する**（mode collapse）。
- これらの語は**「具体を書く労力を省ける便利語」**。「情報が抜けている」と具体的に書く代わりに「情報が落ちる」で済ませられる。AIは具体を持たない（実体験がない）から、この便利語に逃げる。
- 人間も使う語ではある。だが人間は「サーバーが落ちる」のように**実体のある場面**で使う。AIは実体のない抽象にまで被せる。ここがズレる。

> 直し方の大原則：**別のかっこいい語に言い換えるのではなく、比喩をやめて「具体的に何がどうなのか」を書く。**
> 言い換え先もまた偏愛語（解像度→粒度、など）だと意味がない。

---

## 最頻発：AIの偏愛語を"擦り倒す"

### 偏愛する比喩動詞

AIが何にでも被せたがる動詞。**テストは「その動詞、そこで本当に使う？」**。違和感があれば、具体に戻す。

| 動詞 | 本来の居場所（OK） | AIが擦る例（不自然） | 戻し方（具体に） |
|---|---|---|---|
| 落ちる | サーバー・精度・速度が落ちる | 情報が落ちる／視点が落ちる／伝聞が落ちる | 情報が**抜ける・書かれない** |
| 壊れる | 機械・ファイルが壊れる | 仕組みが壊れる／関係が壊れる | 手順が**破綻する**／**うまく回らない**より「何がどうダメか」 |
| 刺さる | （ほぼ常に比喩） | 刺さる文章／言葉が刺さる | 読んだ人が**自分のことだと感じる** |
| 効く | 薬・ブレーキが効く | この一手が効く／設計が効く | 〜すると◯◯が**良くなる** |
| 回る | 歯車・お金が回る | 仕組みが回る／チームが回る | 滞りなく**進む**／担当を**回せる** |
| 描く | 絵を描く | 未来を描く／世界観を描く | どんなものにするか**決める** |
| 紡ぐ | 糸を紡ぐ | 言葉を紡ぐ／物語を紡ぐ | **書く・話す** |
| 立つ | 旗・柱が立つ | 問いが立つ／仮説が立つ | 疑問が**出てくる**／仮説を**置く** |
| 宿る | （比喩） | 思想が宿る／魂が宿る | 削除、または具体 |
| 紐解く | （比喩） | 歴史を紐解く／背景を紐解く | **調べる・振り返る** |
| 浮き彫りにする | （比喩） | 課題を浮き彫りにする | 課題が**はっきりする・わかる** |
| 照らす／照らし出す | 光が照らす | 本質を照らし出す | 削除 |
| 滲む | 液体が滲む | 人柄が滲む | 人柄が**伝わる** |
| 揺さぶる／揺らぐ | 物が揺れる | 常識を揺さぶる | 削除、または「常識と**違う**」 |

> ⚠️ 特に「**落ちる**」は要注意。技術文脈では正しい場面が多い（サーバーが落ちる）が、AIは「情報が落ちる」「精度の話以外でも落ちる」と被せがち。
> 画像で挙がった「AIはこれが落ちる」も、まさにこの擦り倒しの典型。

### 偏愛する抽象名詞・システム翻訳語

AIは、ふつうの概念を**わざわざシステム用語・抽象語に"翻訳"して**被せる。「人生の設計」「思考の解像度」のように、エンジニアリング語を比喩で使うのが癖。

| 偏愛名詞 | 本来の居場所（OK） | AIが擦る例（不自然） | 戻し方 |
|---|---|---|---|
| 設計 | ソフトウェア・建築の設計 | 人生の設計／関係の設計／学びの設計 | どう**進めるか**／**計画** |
| 仕組み | 制度・機構 | 学びを仕組みにする／習慣の仕組み | 具体的な**手順・やること** |
| 構造 | 建物・分子の構造 | 問題の構造／◯◯の構造 | 問題の**中身・関係** |
| 本質 | （哲学用語） | 本質的には／本質はここ | **いちばん大事なのは**◯◯ |
| 解像度 | 画像の解像度 | 課題の解像度を上げる | **具体的に・詳しく** |
| 粒度／濃度／密度／温度 | （物理量） | 議論の粒度／熱量／温度感 | 削除、または具体 |
| 手触り／質感／肌感／体温／血の通った | （物理感覚） | 文章の手触り／血の通った言葉 | 削除、または「読みやすい」等の具体 |
| OS／レイヤー／フェーズ／グラデーション | （技術用語） | 思考のOS／意識のレイヤー | 削除、または平易語 |
| アップデート／インストール／ハック／リファクタリング／チューニング | （IT動作） | 思考をアップデート／人生をハック | **見直す・変える**等の具体 |
| 装置／営み／物語／世界観／まなざし／思想 | （文芸・学術） | 〜という装置／日々の営み／作り手のまなざし | 削除、または具体 |
| 地図／羅針盤／土台／柱 | （物） | キャリアの地図／学びの土台 | 具体（→症状⑬テンプレ比喩） |

### 技術ブログでの線引き（重要）

このパックの読者は技術ブログを書く。だから「設計」「落ちる」「リファクタリング」を**完全に禁止はできない**。線引きはシンプル。

```
✅ OK：実体（物理的・技術的な対象）が本当にある
   - サーバーが落ちる／APIを設計する／コードをリファクタリングする
   - ビルドが通る／テストが落ちる／プロセスが死ぬ（技術スラングとして自然）

❌ NG：実体のない抽象に、比喩として被せている
   - 情報が落ちる／人生を設計する／思考をリファクタリングする
   - 学びの解像度を上げる／キャリアの土台
```

判定基準：**「その対象は、本当に物理的・技術的に"落ちる/設計する"ものか？」**
答えがNoなら、比喩を外して具体に戻す。

### 偏愛する構文

語だけでなく、構文にも偏愛がある。

| 構文 | AIの例 | なぜ臭うか | 直し |
|---|---|---|---|
| 「〇〇は、××だった」（読点＋過去断定） | 「AI臭の記事は、ほぼすべて表層で止まっていた。」 | 観察して発見した風の断定。現在も真なら現在形でいい | 「〜は表層で止まっている。」（現在形・読点を削る） |
| 「〇〇はこれが落ちる」（主語＋これが＋比喩動詞） | 「AIはこれが落ちる。」 | 決め台詞ぶった体言止め＋偏愛動詞 | 「AIはこれができない。」具体に |
| 「〇〇じゃない。△△だ。」 | 「大事なのは言語じゃない。設計だ。」 | カジュアル版の否定→肯定。AIが"決め台詞"で多用（→症状⑧） | 「大事なのは設計です。」と肯定だけ言い切る |
| 「〜じゃなくて、〜」 | 「努力じゃなくて、習慣の問題。」 | 上の口語派生。1記事に何度も出る | 「習慣の問題です。」否定部分を削る |
| 「Aではなく、B」 | 「知識ではなく、翻訳力だ。」 | 対比のための対比（→症状⑧） | AとBを分けて独立に言い切る |
| 「多くの〇〇が〜」 | 「多くのエンジニアが見落としている。」 | 主語不在の一般論で始める | 「自分も見落としていた。」具体・固有に |
| 命題型の体言止め大見出し | H2「設計とは、選択の連続である」 | キャッチコピー臭 | 「設計＝何を捨てるか決めること」内容で |

## その他の崩れ方（動詞・助詞・形容詞の選択ミス）

ここからは系統 2。AIが「近い意味の別の語」で慣用句を壊すパターン。偏愛語ほど頻発しないが、これも不自然さの原因になる。

### 動詞と目的語のミスマッチ

| ❌ AIが書きがち | ○ 自然な結びつき |
|---|---|
| 課題を改善する | 課題を**解決**する（改善するのは「状況・品質」） |
| 目標を成功する | 目標を**達成**する |
| 効果を与える | 効果を**発揮**する／影響を**与える** |
| 影響を発揮する | 影響を**与える・及ぼす** |
| ニーズを叶える | ニーズを**満たす** |
| 期待を満たす | 期待に**応える** |
| 経験を貯める | 経験を**積む** |
| 知識を覚える | 知識を**身につける・得る** |
| 役割を行う | 役割を**果たす・担う** |
| 成長を達成する | 成長を**遂げる** |
| 対策を作る | 対策を**講じる・とる** |
| リスクを解決する | リスクを**減らす・回避する** |
| バグを解決する | バグを**修正する**／エラーを**解決する** |
| パフォーマンスを強化する | パフォーマンスを**向上させる** |
| メモリを削減する | メモリ使用量を**減らす・抑える** |
| コストを節約する | コストを**削減する・抑える** |

### 自動詞・他動詞の取り違え（が・を）

「向上・発生・変化」などの漢語サ変動詞は、自動詞か他動詞かが曖昧。他動詞にしたいなら「〜させる」を付ける。

| ❌ | ○ |
|---|---|
| 速度を向上する | 速度**が**向上する／速度**を**向上**させる** |
| エラーを発生する | エラー**が**発生する／エラー**を**発生**させる** |
| 値を変化する | 値**が**変化する／値**を**変化**させる** |

### 技術スラングの自然な動詞（覚えておくと"落ちる"の誤用が減る）

実体のある技術現象には、自然な和語動詞が決まっている。これを知っていると、偏愛動詞の擦り倒しと区別できる。

| 主語 | ○ 自然な動詞 |
|---|---|
| サーバー・プロセス・アプリ | 落ちる／死ぬ／立ち上がる／立てる |
| ビルド・テスト・CI | 通る／落ちる／こける |
| バグ | 踏む／潰す／（が）紛れ込む |
| データ・値 | 飛ぶ／消える／化ける（文字化け） |
| 処理・通信 | 詰まる／止まる／固まる／流れる／走る |
| メモリ | 食う／溢れる／漏れる（リーク） |
| ログ | 出る／吐く／流れる |
| フラグ | 立つ／立てる／寝かせる |
| 設定・キャッシュ | 効く／効かない／残る |
| レイアウト・画面 | 崩れる／ずれる／固まる |

> これらは**実体があるからOK**。「情報が落ちる」「議論が詰まる（比喩）」のように実体のない所へ持ち出すと擦り倒しになる。

### 形容詞・連体修飾と名詞のミスマッチ

| ❌ | ○ |
|---|---|
| 濃い理解 | 深い理解 |
| 高い課題 | 大きな課題 |
| 大きい精度 | 高い精度 |
| 濃い相関 | 強い相関／高い相関 |
| ハードルが大きい | ハードルが高い |

「理解は深い、精度は高い、相関は強い、課題は大きい」。付く形容詞が決まっている。

### 副詞と述語の程度ミスマッチ

| ❌ | ○ |
|---|---|
| 大幅に解決する | 大幅に**改善する**／完全に**解決する** |
| 劇的に達成する | 劇的に**向上する**／確実に**達成する** |

「大幅に・劇的に」は変化の動詞（改善・向上）に、「完全に・確実に」は到達点の動詞（解決・達成）に付く。

### 重言（意味の重複）

| ❌ 重言 | ○ |
|---|---|
| まず最初に | まず／最初に |
| 各〜ごとに | 〜ごとに／各〜 |
| 約〜程度 | 約〜／〜程度 |
| 違和感を感じる | 違和感を覚える |
| throwを投げる | 例外を投げる（throw＝投げる） |
| inputを入力する | 入力する |

技術系で特に出るのが**「英単語＋その和訳」の重複**。片方にする。

### 翻訳調（英語の直訳的結びつき）

| ❌ 翻訳調 | ○ 自然な日本語 |
|---|---|
| 影響を作る（make an impact） | 影響を与える |
| 決定を作る（make a decision） | 決定する／判断する |
| 違いを作る（make a difference） | 違いを生む／効果がある |
| 質問を持っている（have a question） | 質問がある |
| 高いクオリティを持つ | 質が高い |

## 直し方の手順

1. **まず偏愛語を疑う**：落ちる・壊れる・刺さる・効く・回る・設計・仕組み・構造・本質・解像度・手触り…が出たら立ち止まる
2. **「その対象に実体があるか」を判定**：サーバー（実体）→OK、情報（抽象）→NG
3. **NGなら具体に戻す**：別の比喩に言い換えない。「何がどうなのか」を書く
4. **構文の癖を直す**：「〜は、〜だった」→現在形に。「AではなくB」→分ける
5. **動詞・助詞・形容詞のミスを正す**（後半の表を参照）
6. **重言・翻訳調を消す**

## AIに直させるプロンプト

```
以下の文章から、AIが多用しがちな「偏愛語の被せ」を直してください。意味と事実は変えないこと。

【偏愛動詞】落ちる・壊れる・刺さる・効く・回る・描く・紡ぐ・立つ・宿る・紐解く・浮き彫りにする
【偏愛名詞】設計・仕組み・構造・本質・解像度・粒度・手触り・質感・温度・OS・レイヤー・グラデーション・装置・営み・まなざし・アップデート・ハック・リファクタリング（比喩用法のとき）
【偏愛構文】「〇〇は、××だった」過去断定 ／「AではなくB」対比 ／「多くの〇〇が」一般論始まり

ルール：
- これらの語が「実体のある技術対象」に付いている場合（サーバーが落ちる、APIを設計する）は残す
- 「実体のない抽象」に比喩で被せている場合（情報が落ちる、人生を設計する）は、別の比喩に言い換えず、具体的に何がどうなのかに書き直す
- 「〇〇は、××だった」は、現在も真なら現在形にする

修正箇所を変更前→変更後で一覧にし、その後に直した全文を出してください。
```

## セルフチェック

### 偏愛語（最優先）
- [ ] 「落ちる・壊れる・刺さる・効く・回る」が、実体のない抽象に被さっていないか
- [ ] 「設計・仕組み・構造・本質・解像度・手触り」を比喩で乱用していないか（実体があればOK）
- [ ] システム用語・横文字（OS・レイヤー・アップデート・ハック）を人間の活動に被せていないか
- [ ] 「〇〇は、××だった」の観察発見ふう過去断定が連発していないか
- [ ] 偏愛語を直すとき、別の偏愛語に言い換えず「具体」に戻せているか

### 選択ミス
- [ ] 「課題・目標・効果・リスク」に付く動詞は正しいか
- [ ] 漢語サ変動詞の助詞（が／を）は合っているか（速度が向上する／速度を向上させる）
- [ ] 形容詞と名詞が合っているか（理解は深い、精度は高い）
- [ ] 重言（まず最初に／throwを投げる）が無いか
- [ ] 翻訳調（影響を作る）が無いか
