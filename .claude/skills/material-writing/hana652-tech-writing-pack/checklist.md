# 公開前セルフチェック（1枚）

記事を公開する前に、これだけ見れば最低限OKという1枚。
詳細は各ガイドへ。チェックが付かない項目があれば、対応するガイドに戻る。

---

## 設計（→ 01-seo-for-dev-blogs.md）
- [ ] 「誰の、どんな検索に答える記事か」が1行で言える
- [ ] タイトルにキーワードが含まれ、具体的（「徹底解説」「完全ガイド」を使っていない）
- [ ] リード文の3行で「何が解決するか」が伝わる
- [ ] 検索意図のタイプ（やり方/エラー/比較/概念）に構成が合っている

## 構成（→ 02-article-structures.md）
- [ ] 結論先行になっている（答え → 理由 → 詳細）
- [ ] 1見出し1トピック。見出しを読むだけで流れがわかる
- [ ] 見出しが「作業の実況」でなく「内容」で書かれている
- [ ] コードはコピペで動く形（import・前提を省略していない／実行結果を示した）

## AI臭（→ 03-anti-ai.md）
- [ ] 書き手の態度が感じられる（全部同じ温度になっていない）
- [ ] 各セクションに言い切りの文が最低1つある
- [ ] ヘッジ（〜でしょう/かもしれません）が過剰でない
- [ ] メタ発言（〜を解説します/見ていきましょう）を削った
- [ ] 章境界アナウンス（ここから〜/ここまで〜してきました/次は〜）が無い
- [ ] テンプレ比喩（地図/土台/車の両輪/潤滑油/スパイス…）を使っていない
- [ ] 「〜じゃない。〜だ。」の対比構文が1記事に2回以上出ていない
- [ ] 文長に緩急がある（短文と長文が混ざっている）
- [ ] 同じ語尾が3回以上連続していない
- [ ] 一文を短くしすぎて主語・目的語を削っていない（ブツ切り回避）
- [ ] ですます/である調が統一されている（常体落ちが無い）

## コロケーション（→ 04-collocation.md）
- [ ] 「課題・問題・エラー・目標・効果」に付く動詞が正しい
- [ ] 漢語サ変動詞の助詞（が/を）が合っている（速度が向上する／速度を向上させる）
- [ ] 形容詞と名詞が合っている（理解は深い、精度は高い）
- [ ] 重言が無い（まず最初に／throwを投げる／約〜程度）
- [ ] 翻訳調が無い（影響を作る → 影響を与える）

## 正確さ（→ 05-technical-writing.md）
- [ ] 60字超の長文を分割した
- [ ] 用語が記事全体で統一されている
- [ ] 数量・期間が具体的な数値（早めに → 1週間以内に）
- [ ] AND/ORが明確（「AやB」で曖昧にしていない）
- [ ] ですます/である調が混在していない

## 記号（→ 03-anti-ai.md）
- [ ] `**太字**` のアスタリスク残りが無い（コードブロック外）
- [ ] `—`（em-dash）`──`（罫線）を使っていない
- [ ] `：`+半角スペース、`／`での概念並列を使っていない

## 機械チェック
- [ ] `python3 scripts/ai-smell-lint.py 記事.md` を実行し、FATALがゼロ
- [ ] warnは目視で要否を判断した

## 最後の一押し（人間にしかできない）
- [ ] 自分の実体験・判断・動作確認した結果が入っている
- [ ] コードを実際に動かして確認した
- [ ] 「答えを書かずに問いで終わる」箇所が1〜2箇所ある（任意）
