## 2. 教材ビジョン

この教材で作るものは、個人用の練習アプリではなく、複数人で使う **チームタスク管理アプリ** である。

学習者が 30 日後に見せる成果物は、次の状態を満たす。

- ログインできる。
- プロジェクトを作れる。
- タスクを追加、更新、削除できる。
- 担当者やメモを付けられる。
- 進み具合を一覧、数字カード、グラフで見られる。
- プロフィールや管理者画面まで含めて、ひとつのプロダクトとして見せられる。
- 最後に公開 URL で友人へ共有できる。

この教材の対象者は、**完全初心者** である。

- プログラミング未経験でも始められることを前提にする。
- JavaScript の基礎知識が薄くても、画面を見ながら進められることを優先する。
- 「既存コードを読み解ける人」ではなく、「まだ読めないからこそ画面の変化で理解したい人」を想定する。
- したがって、技術用語の理解より先に、機能が目の前で増える体験を置く。

守る原則は 3 つだけに絞る。

1. 毎日「今日作ったもの」を画面で見せる。スクリーンショットは必須とし、文章だけで終わらせない。
2. 技術単位ではなく機能単位で進む。タイトルは必ず「何ができるようになるか」を表す。
3. 難易度は Day 01=1.0 から Day 30=3.5 まで単調増加にし、途中で急坂を作らない。

この原則により、学習者の心理的ゴールは毎日明確になる。

- 今日は何を作るのか。
- 終わったら何を友人に見せられるのか。
- 次の日はその機能のどこを一段だけ広げるのか。

技術は捨てない。
ただし、技術は **機能を作るための裏方** として後ろに置く。

- 用語は必要な分だけ出す。
- 背景説明は `details` または発展へ逃がす。
- 成功条件、スクリーンショット、戻り先は常に本線に残す。

教材全体の完成イメージは次の一文で定義する。

**「完全初心者が、30日かけて、毎日ひとつずつ見せられる機能を積み上げ、チームタスク管理アプリを公開できる教材」**
