## 実装内容

### Part 1: 必要なツールの確認(20分)

#### Step 1.1: Node.jsのバージョン確認(所要時間:5分)

**このステップで学ぶこと**: Node.jsが正しくインストールされているかを確認する方法。

**なぜ必要?**: TaskAppはNode.js v25.x が必要です（`.mise.toml` で v25.6.1 を指定）。違うバージョンでは `npm install` が失敗します。料理に例えると、レシピに「オーブン200度」と書いてあるのに、オーブンがなければ料理が作れないのと同じです。

**コードの仕組み解説**:
- `node --version`: インストールされているNode.jsのバージョンを表示します
- `v25.6.1`のように表示されれば成功です

このプロジェクトは [mise](https://mise.jdx.dev/) でバージョン管理しています。以下のコマンドでNode.jsをセットアップしてください:

```bash
# miseのインストール（未導入の場合）
curl https://mise.run | sh

# プロジェクトディレクトリで実行
mise install
node --version
```

**確認方法**:
1. ターミナルを開く(VS Codeなら`Ctrl+@`または`Cmd+@`)
2. 上記コマンドを実行
3. `v25.6.1` が表示されることを確認

**よくあるエラー**:
- エラー: `command not found: node`
  - 解決: `mise install` をプロジェクトディレクトリで実行してください。miseが未導入の場合は `curl https://mise.run | sh` でインストール

---

#### Step 1.2: npmのバージョン確認(所要時間:5分)

**このステップで学ぶこと**: npm(Node Package Manager)の確認方法。

**なぜ必要?**: npmはJavaScriptのパッケージ(ライブラリ)を管理するツールです。Node.jsと一緒にインストールされます。料理で例えると、食材を仕入れる市場のようなものです。

**コードの仕組み解説**:
- `npm --version`: npmのバージョンを表示します

以下のコマンドを**そのまま実行**してください:

```bash
npm --version
```

**確認方法**:
1. バージョン番号(例: `10.x.x`)が表示されれば成功

**よくあるエラー**:
- エラー: バージョンが古い
  - 解決: `npm install -g npm`で最新版に更新できます

---

#### Step 1.3: PostgreSQLの準備(所要時間:10分)

**このステップで学ぶこと**: データベースの準備方法。

**なぜ必要?**: TaskAppはPostgreSQLというデータベースにデータを保存します。データベースは「情報を整理して保管する倉庫」のようなものです。

**選択肢1: Docker Desktop(推奨)**

Dockerがインストールされている場合、以下のコマンドでPostgreSQLを起動できます:

```bash
docker --version
```

**選択肢2: クラウドデータベース**

ローカルにPostgreSQLをインストールしたくない場合、以下のサービスが無料で使えます:
- [Supabase](https://supabase.com/) - 無料枠あり
- [Neon](https://neon.tech/) - 無料枠あり

**確認方法**:
1. Docker Desktop: アプリケーションが起動していることを確認
2. クラウドDB: 接続URLを取得できていることを確認

### Part 2: プロジェクトのセットアップ(30分)

#### Step 2.1: プロジェクトのクローン(所要時間:5分)

**このステップで学ぶこと**: GitHubからプロジェクトをダウンロードする方法。

**なぜ必要?**: 「クローン」とは、リモートのリポジトリ(コードの保管場所)を自分のパソコンにコピーすることです。図書館で本を借りてくるようなイメージです。

**コードの仕組み解説**:
- `git clone`: リポジトリをコピーするコマンド
- `cd task-app`: task-appディレクトリに移動

以下のコマンドを**そのまま実行**してください:

```bash
git clone https://github.com/kouiso/task-app.git
cd task-app
```

**確認方法**:
1. `task-app`フォルダが作成されていることを確認
2. `ls`(Mac/Linux)または`dir`(Windows)でファイル一覧を確認

**よくあるエラー**:
- エラー: `git: command not found`
  - 解決: Gitがインストールされていません。[Git公式サイト](https://git-scm.com/)からインストールしてください

---

#### Step 2.2: 依存パッケージのインストール(所要時間:10分)

**このステップで学ぶこと**: プロジェクトに必要なライブラリをインストールする方法。

**なぜ必要?**: `package.json`には、プロジェクトで使うライブラリのリストが書かれています。`npm install`を実行すると、これらのライブラリが`node_modules`フォルダにダウンロードされます。料理で例えると、レシピに書かれた材料をすべて買い揃えるようなものです。

**コードの仕組み解説**:
- `npm install`: package.jsonに記載されたすべてのパッケージをインストール
- `node_modules/`: インストールされたパッケージが格納されるフォルダ
- `package-lock.json`: インストールされたパッケージの正確なバージョンを記録

以下のコマンドを**そのまま実行**してください:

```bash
npm install
```

**確認方法**:
1. `node_modules`フォルダが作成されていることを確認
2. エラーなく完了すること(警告は無視して構いません)

**よくあるエラー**:
- エラー: `EACCES permission denied`
  - 解決: 管理者権限が必要な場合があります。`sudo npm install`を試してください(Mac/Linux)

---

#### Step 2.3: 環境変数の設定(所要時間:10分)

**このステップで学ぶこと**: 環境変数ファイルの作成方法。

**なぜ必要?**: 環境変数は、データベースのパスワードやAPIキーなど、公開してはいけない情報を安全に管理するための仕組みです。`.env`ファイルはGitにコミットされないため、秘密情報を安全に保管できます。

**コードの仕組み解説**:
- `.env.example`: 環境変数のテンプレート(サンプル値が入っている)
- `.env`: 実際に使用する環境変数ファイル(自分で作成)
- `DATABASE_URL`: PostgreSQLへの接続情報

以下のコマンドを**そのまま実行**してください:

```bash
cp .env.example .env
```

次に、`.env`ファイルを開いて、`DATABASE_URL`を設定します:

```bash
# filepath: .env
# データベース接続URL
# Docker使用時
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/taskapp?schema=public"

# 認証用のシークレットキー(32文字以上のランダムな文字列)
AUTH_SECRET="your-super-secret-key-change-this-in-production"
```

**確認方法**:
1. `.env`ファイルが作成されていることを確認
2. `DATABASE_URL`が正しく設定されていることを確認

**よくあるエラー**:
- エラー: 接続できない
  - 解決: PostgreSQLが起動しているか確認してください

---

#### Step 2.4: データベースのセットアップ(所要時間:5分)

**このステップで学ぶこと**: Prismaを使ったデータベースの初期化方法。

**なぜ必要?**: Prismaは、TypeScriptからデータベースを操作するためのツール(ORM)です。`prisma migrate dev`を実行すると、`prisma/schema.prisma`に定義されたテーブルがデータベースに作成されます。

**コードの仕組み解説**:
- `npx prisma migrate dev`: マイグレーション(データベースの構造変更)を実行
- `--name init`: マイグレーションに「init」という名前をつける
- `npx prisma generate`: Prisma Clientを生成(TypeScriptから使うためのコード)

Dockerを使用する場合は、まずコンテナを起動します:

```bash
docker compose up -d
```

次に、データベースをセットアップします:

```bash
npx prisma migrate dev --name init
```

**確認方法**:
1. `prisma/migrations`フォルダにマイグレーションファイルが作成されていること
2. エラーなく完了すること

**よくあるエラー**:
- エラー: `Can't reach database server`
  - 解決: PostgreSQLが起動しているか確認してください
- エラー: `Environment variable not found: DATABASE_URL`
  - 解決: `.env`ファイルが正しく作成されているか確認してください

### Part 3: 動作確認(10分)

#### Step 3.1: シードデータの投入(所要時間:5分)

**このステップで学ぶこと**: テスト用データをデータベースに投入する方法。

**なぜ必要?**: シードデータは、開発時に使うサンプルデータです。空のデータベースでは動作確認が難しいため、最初からデータを入れておきます。

**コードの仕組み解説**:
- `npx prisma db seed`: `prisma/seed.ts`を実行してデータを投入

以下のコマンドを**そのまま実行**してください:

```bash
npx prisma db seed
```

**確認方法**:
1. 「Seeding finished」というメッセージが表示されること
2. エラーなく完了すること

---

#### Step 3.2: 開発サーバーの起動(所要時間:5分)

**このステップで学ぶこと**: Next.jsの開発サーバーを起動する方法。

**なぜ必要?**: 開発サーバーを起動すると、ブラウザでアプリケーションを確認できます。コードを変更すると自動的に画面が更新される「ホットリロード」機能も使えます。

**コードの仕組み解説**:
- `npm run dev`: package.jsonの`scripts.dev`に定義されたコマンドを実行
- 内部では`next dev`が実行され、開発サーバーが起動します
- デフォルトでは`http://localhost:3000`でアクセスできます

以下のコマンドを**そのまま実行**してください:

```bash
npm run dev
```

**確認方法**:
1. ターミナルに「Ready」と表示されること
2. ブラウザで`http://localhost:3000`を開く
3. TaskAppのログイン画面が表示されること

**よくあるエラー**:
- エラー: `Port 3000 is already in use`
  - 解決: 他のアプリケーションがポート3000を使用しています。`npm run dev -- -p 3001`で別のポートを使用できます
- エラー: 画面が真っ白
  - 解決: ブラウザのコンソール(F12)でエラーを確認してください
- エラー: `P1001: Can't reach database server`
  - 解決: PostgreSQL が起動しているか確認し、`docker-compose up -d db` を再実行してください。`DATABASE_URL` のホスト名が合っているかも確認します。

- **Prisma**: TypeScriptからデータベースを操作するためのORM
