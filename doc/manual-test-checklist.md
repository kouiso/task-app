# 手動テストチェックリスト

`doc/manual-test-checklist.csv` は task-app の主要機能を手動で確認するためのチェックリスト（全152ケース、うちレスポンシブ41ケース）。
スプレッドシート（Google Sheets / Excel / Numbers）にそのまま取り込んで利用する。

## 使い方

### Google Sheets

1. Google Sheets で新規シートを開く
2. ファイル → インポート → アップロード → `doc/manual-test-checklist.csv` を選択
3. インポート場所「現在のシートを置換する」、区切り文字「カンマ」 → インポート
4. 「ステータス」「実行結果」「備考」「実施日」列を順次埋めていく

### Excel

1. データ → テキストまたは CSV から → `doc/manual-test-checklist.csv` を選択
2. 元のファイル「UTF-8」、区切り記号「カンマ」 → 読み込み

### TIPS

- 「ステータス」列は `Pass` / `Fail` / `Skip` / `Blocked` のいずれかを入力する運用を推奨
- 条件付き書式で `Fail` を赤、`Pass` を緑にすると一覧性が上がる
- フィルタを有効化してカテゴリ別に絞り込んで実行するとラク

## 列定義

| 列 | 内容 |
|---|---|
| ID | テストID（カテゴリ別の連番。T0xx=認証, T1xx=ダッシュボード, T2xx=タスク, T3xx=マイタスク, T4xx=プロジェクト, T5xx=検索, T6xx=レポート, T7xx=週次, T8xx=ユーザー管理, T9xx=プロフィール, TAxx=レイアウト, TBxx=セキュリティ, **TCxx=レスポンシブ（画面幅別）**）|
| カテゴリ | 機能カテゴリ |
| 画面 | 対象画面の URL またはコンポーネント位置 |
| テスト項目 | 何を確認するか |
| 前提条件 | 実行前に必要な状態 |
| 手順 | 操作の具体ステップ |
| 期待結果 | 期待される動作・表示 |
| 実行結果 | 実際の結果（テスト時に記入） |
| ステータス | Pass / Fail / Skip / Blocked |
| 備考 | エラー詳細やスクショリンクなど |
| 実施日 | YYYY-MM-DD |

## 推奨フロー

1. 機能改修後、影響範囲のカテゴリだけ抜粋して実施
2. リリース前は全件実施
3. `Fail` が出たら備考に GitHub Issue 番号 or スクショパスを記入

## メンテナンス

新機能追加・改修時は本CSVに該当ケースを追記する。特に **回帰確認用** のケース（「○○(回帰)」と表記）は過去バグの再発防止策なので削除しないこと。

## レスポンシブテストの実施ガイド (TCxx)

### 推奨検証幅

| 幅 | デバイス相当 | 検証ポイント |
|---|---|---|
| 375px | iPhone SE / 13 mini | サイドバー隠れる / ハンバーガー表示 / タッチターゲット |
| 768px | iPad 縦 / 中間域 | サイドバー出現境界 (md ブレイクポイント) |
| 1024px | ラップトップ小 / iPad 横 | サイドバー幅変化 (lg ブレイクポイント) |
| 1280px | デスクトップ標準 | フル表示 / 横スクロール無し |

### Chrome DevTools での切替手順

1. F12 で DevTools を開く
2. デバイスツールバー（Cmd+Shift+M / Ctrl+Shift+M）を有効化
3. 上部の寸法入力に `375` `768` `1024` `1280` を直接入力して切替
4. ページを再読込してレイアウトを確認
