# Day 01: 開発環境を整えて、初めてのアプリを動かそう

## 今日のゴール

今日の終わりには、以下のことができるようになります。

- Node.jsとnpmがインストールされ、`node --version` でバージョンが確認できる
- VS Codeでプロジェクトを開き、コードを編集できる環境が整う
- ブラウザで `http://localhost:3000` にアクセスしてタスク管理アプリが動く

開発の第一歩は「環境構築」です。料理を始める前に「キッチンを整える」ように、プログラミングを始める前に「開発環境を整える」必要があります。

## よくあるつまずきと対処法

環境構築は初学者がつまずきやすい場所です。先に対処法を知っておくと安心です。

| つまずきポイント | よくある原因 | 対処法 |
|-----------------|-------------|--------|
| `command not found` | ターミナルを再起動していない | ターミナルを一度閉じて開き直す |
| パスが通らない | インストール後の設定漏れ | PCを再起動する |
| バージョンが古い | 別バージョンが残っている | 古いバージョンをアンインストールしてから再インストール |
| 権限エラー | 管理者権限がない | Windowsは「管理者として実行」、Mac/Linuxは `sudo` |
| Windowsで動かない | WSL2が必要な場合がある | Day02でDocker導入時に詳しく説明 |

## 今日学ぶツール

今日は3つの新しいツールを学びます。

**Node.js（ノードジェイエス）** は、JavaScriptをパソコン上で動かすための土台です。電気を通す「コンセント」のようなもので、これがないとアプリに「電気」が通りません。

**npm（エヌピーエム）** は、パッケージ管理ツールです。図書館の「司書さん」のように、必要なプログラム部品を集めてきてくれます。

**VS Code（ブイエスコード）** は、コードを書くための専用エディタです。メモ帳でも書けますが、VS Codeは「補助輪付き自転車」のように開発を助けてくれます。

```mermaid
flowchart LR
    A[あなたのPC] --> B[VS Code]
    B --> C[Node.js]
    C --> D[Next.js App]
    D --> E[ブラウザで確認]
```

> **補足**: 今日はデータベース（Docker）の導入は行いません。Day02で詳しく学びます。今日はまず「アプリが動く」ところまで集中しましょう。

## Step 1: Node.jsをインストールする（7分）

Node.jsをインストールすると、npm（パッケージ管理ツール）も一緒にインストールされます。

### 操作手順

1. 公式サイト https://nodejs.org/ にアクセスする
2. **「LTS」と書かれた緑色のボタン**をクリックする（v24.x）
3. ダウンロードしたファイルを実行し、指示に従ってインストールする

インストール完了後、ターミナルを開いてバージョンを確認します。

### ターミナルの開き方

| OS | 開き方 |
|----|--------|
| Windows | 「スタート」→「PowerShell」と検索 |
| Mac | 「Spotlight（⌘+Space）」→「ターミナル」と検索 |
| Linux | Ctrl + Alt + T |

```bash
# filepath: ターミナル
# Node.jsのバージョンを確認する
node --version
```

**確認ポイント:**
- `v24.11.1` や `v24.12.0` のような表示が出る
- 数字の最初が `24` であればOK

```bash
# filepath: ターミナル
# npmのバージョンを確認する
npm --version
```

**確認ポイント:**
- `10.9.0` のような表示が出る
- 数字の最初が `10` 以上であればOK

**うまくいかない場合:**
- `command not found: node` → ターミナルを閉じて再度開く
- バージョンが古い → 公式サイトから最新版を再インストール

## Step 2: VS Codeをインストールする（5分）

VS Code（Visual Studio Code）は、Microsoftが開発した無料のコードエディタです。

### 操作手順

1. 公式サイト https://code.visualstudio.com/ にアクセスする
2. 「Download」ボタンをクリックする
3. ダウンロードしたファイルを実行する

### 拡張機能をインストールする

VS Codeを起動したら、左側のサイドバーにある「四角が4つ並んだアイコン」をクリックして拡張機能画面を開きます。

以下の拡張機能を検索してインストールしてください。

- **Japanese Language Pack**（検索：japanese）→ メニューを日本語化
- **Prisma**（検索：prisma）→ データベース定義のハイライト
- **Tailwind CSS IntelliSense**（検索：tailwind）→ スタイルの補完

**確認ポイント:**
- VS Codeが起動し、左側サイドバーにアイコンが表示される
- 拡張機能インストール後、VS Codeを再起動するとメニューが日本語になる

【スクリーンショット: VS Codeの拡張機能インストール画面】

**うまくいかない場合:**
- 拡張機能が見つからない → 検索キーワードを正確に入力
- 日本語にならない → VS Codeを完全に閉じて再度開く

## Step 3: プロジェクトをクローンする（5分）

GitHub（ギットハブ）からプロジェクトのコードをダウンロードします。「クローン」とは、リポジトリ（コードの保管庫）の完全なコピーを作ることです。

### 操作手順

```bash
# filepath: ターミナル
# 作業フォルダに移動する（例：デスクトップ）
cd ~/Desktop
```

**確認ポイント:**
- エラーが出ない
- プロンプト（入力待ち状態）に戻る

```bash
# filepath: ターミナル
# GitHubからプロジェクトをクローンする
git clone https://github.com/your-org/task-app.git
```

**確認ポイント:**
- `Cloning into 'task-app'...` と表示される
- 最後に `done.` と表示される

```bash
# filepath: ターミナル
# プロジェクトフォルダに移動する
cd task-app
```

**確認ポイント:**
- エラーが出ない
- プロンプトが `task-app` を含む表示に変わる

**うまくいかない場合:**
- `git: command not found` → https://git-scm.com/ からGitをインストール
- `already exists` → すでにクローン済み。`cd task-app` でフォルダに移動

---

## Step 4: VS Codeでプロジェクトを開く（3分）

クローンしたプロジェクトをVS Codeで開きます。

### 操作手順

```bash
# filepath: ターミナル
# VS Codeでプロジェクトを開く
code .
```

**確認ポイント:**
- VS Codeが起動する
- 左側のエクスプローラーに `src/`、`prisma/`、`package.json` などが表示される

【スクリーンショット: VS Codeでプロジェクトを開いた状態】

**うまくいかない場合:**
- `code: command not found` → VS Codeを開き、コマンドパレット（F1）で「Shell Command: Install 'code' command in PATH」を実行
- 手動で開く → VS Codeを起動し、「ファイル」→「フォルダを開く」から `task-app` フォルダを選択

---

## Step 5: 依存パッケージをインストールする（5分）

npmを使って、プロジェクトに必要なパッケージをインストールします。「司書さんが必要な本を全部集めてくる」作業です。

### 操作手順

```bash
# filepath: ターミナル
# 依存パッケージをインストールする
npm install
```

**確認ポイント:**
- 最後に `added XXX packages in YYs` のような表示が出る
- `node_modules` フォルダが作成されている

### 作成されるフォルダの説明

| フォルダ/ファイル | 役割 |
|------------------|------|
| `node_modules/` | ダウンロードしたパッケージの実体（数千個のファイル） |
| `package-lock.json` | パッケージのバージョンを固定するファイル |

**うまくいかない場合:**
- `WARN` で始まる警告 → 軽微な警告なので無視してOK
- `ERR!` で始まるエラー → `rm -rf node_modules` を実行してから再度 `npm install`

---

## Step 6: 環境変数ファイルを作成する（3分）

環境変数（かんきょうへんすう）とは、アプリの設定情報をまとめたものです。パスワードやAPIキーなど、コードに直接書きたくない情報を外部ファイルに保存します。

### 操作手順

```bash
# filepath: ターミナル
# サンプルファイルをコピーして.envを作成する
cp .env.example .env
```

**確認ポイント:**
- エラーが出ない（何も出力されないのが正常）
- `ls -la .env` で `.env` ファイルが存在することを確認

**うまくいかない場合:**
- `No such file` → `.env.example` がない場合は `git pull` で最新化
- `.env` が見えない → 隠しファイルなので `ls -la` で表示

---

## Step 7: 開発サーバーを起動する（5分）

いよいよ、アプリケーションを起動してブラウザで確認します。

> **注意**: Day01ではデータベースを使わない簡易起動を行います。フル機能（ログイン等）はDay02でDocker導入後に確認します。

### 操作手順

```bash
# filepath: ターミナル
# 開発サーバーを起動する
npm run dev
```

**確認ポイント:**
- `▲ Next.js 15.x.x` と表示される
- `✓ Ready in XXXms` と表示される
- ターミナルは開いたままにする（閉じるとサーバーが停止）

【スクリーンショット: npm run dev 実行後のターミナル画面】

### ブラウザでアクセスする

1. ブラウザを開く（Chrome推奨）
2. アドレスバーに `http://localhost:3000` と入力する
3. Enterキーを押す

【スクリーンショット: アプリの初期画面が表示された状態】

**確認ポイント:**
- アプリの画面が表示される
- エラー画面（赤い背景）が出ていない

**うまくいかない場合:**
- 白い画面が表示 → ターミナルで `npm run dev` の状態を確認
- `EADDRINUSE: port 3000` → 下記コマンドでポートを解放

```bash
# filepath: ターミナル（Mac/Linux）
# ポート3000を使っているプロセスを終了する
kill $(lsof -t -i:3000)
```

**確認ポイント:**
- エラーが出ない
- 再度 `npm run dev` が実行できる
