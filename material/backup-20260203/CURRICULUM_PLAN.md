# task-app 30日間教材 完全作成計画

## 📋 概要

| 項目 | 内容 |
|------|------|
| **作成対象** | Day 1〜30 の教材（既存Day 1-3は改訂） |
| **対象者** | 文法は学んだがアプリ作成は初めての人 |


### 3. デプロイは最初と最後

```
Day 1-3           Day 4          Day 5-28          Day 29-30
┌──────────┐    ┌────────┐    ┌──────────┐      ┌──────────┐
│ 環境構築  │ ─→ │ 初回   │ ─→ │  機能    │  ─→  │ 最終     │
│ Git/GitHub│    │ デプロイ│    │  開発    │      │ デプロイ │
│ 動作確認  │    │   🚀   │    │  学習    │      │   🎓    │
└──────────┘    └────────┘    └──────────┘      └──────────┘
                    │                                │
                    └────── push = 自動反映 ─────────┘
```

**理由**:
- 最後だけだと失敗時の原因特定が困難
- 最後に大きなエネルギー消耗を避ける
- 「push = 自動デプロイ」の感覚を早期に習得

## 🛠️ PDF化の仕組み

### ツール選定: `md-mermaid-to-pdf`

| 比較項目 | md-mermaid-to-pdf | Pandoc+XeLaTeX | Marp |
|---------|-------------------|----------------|------|
| セットアップ | npm install のみ | texlive必要 | npm install |
| テキスト流れ | 連続的 | 連続的 | ページ単位 |
| Mermaid対応 | 組み込み済み | 別途処理必要 | 組み込み |
| CSS制御 | 完全自由 | LaTeX制約あり | テーマベース |
| 日本語 | Chrome処理で自然 | XeLaTeX設定必要 | Chrome処理 |
| **推奨度** | **★★★** | ★★ | ★（長文不向き） |

### ホットリロード環境

編集中にリアルタイムでプレビュー確認:
```bash
# HTML + Live Server
npx chokidar-cli "material/**/*.md" -c "./scripts/generate_html.sh {path}" &
npx live-server output/
```

## 🚫 redmine-cloneの反省点と対策

| 問題 | 原因 | 対策 |
|------|------|------|
| ステップ飛ばし | AIが「同様に」で省略 | 1ステップ1ファイル変更、全コード明示 |
| 文字ばかり | 解説が長文 | 最大3文ルール、表・Mermaid図で置き換え |
| 教科書的 | 説明→コードの一方通行 | コード先行 + 表形式解説 |
| 1回の反映がデカい | コード変更量が巨大 | コードブロック上限25行 |
| 初心者にわかりづらい | 前提知識の暗黙要求 | 新概念は必ず例え話＋比較表 |

## 🔄 制作フロー

### Step 1: PDF生成基盤の構築
1. `npm install --save-dev md-mermaid-to-pdf`
2. カスタムCSS作成
3. 品質チェックスクリプト配置

### Step 2: 全Day骨格ファイル作成
1. Day 1-30の全ファイルをテンプレート骨格で作成
2. ステップ一覧表だけ入った状態

### Step 3: Phase単位で中身を埋める
1. 対象機能の仕様確認
2. コードを25行以下ブロックに分割計画
3. テンプレートに沿って全ステップ記述
4. 表・Mermaid図追加
5. 品質チェックスクリプト実行

### Step 4: PDF生成・確認
```bash
./scripts/generate_pdf.sh material/30days-curriculum/dayXX_xxx.md
```
