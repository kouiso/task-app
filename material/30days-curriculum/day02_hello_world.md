### localhost:3000 って何？

**localhost**は「あなた自身のパソコン」という意味です。**3000**は「ポート番号」といって、「どの出入り口を使うか」を指定しています。

> 💡 **例え話**: マンションの部屋番号
>
> - **localhost** = あなたの住んでいるマンション（建物自体）
> - **3000** = 部屋番号（301号室、みたいなもの）
>
> つまり、`http://localhost:3000`は「あなたのパソコンの3000番ポートで動いているアプリにアクセスする」という意味です。
>
> インターネット上の他の人はアクセスできません。あなただけが見られる「自分専用のWebサイト」です。

### Step 1: Gitのインストールと確認

**🔰 初心者向け解説**:

Gitは、「コードのバージョン管理システム」です。GitHubからコードをダウンロードするために必要です。

> 💡 **例え話**:
> Gitは「タイムマシン付きのUSBメモリ」のようなものです。
>
> - **普通のUSBメモリ**: ファイルを保存できるだけ
> - **Git**: ファイルの「履歴」も保存できる
>   - 「昨日の状態に戻す」ができる
>   - 「誰が、いつ、何を変更したか」がわかる
>   - 「複数人で同じファイルを編集」できる
>
> 今日は、GitHubに保存されているコードを、Gitを使ってダウンロード（クローン）します。

**💻 インストール手順**:

#### Gitがすでにインストールされているか確認

まず、昨日インストールしたNode.jsと一緒にGitもインストールされている可能性があります。確認しましょう。

コマンドプロンプト（Windows）またはターミナル（Mac）を開いて：

```bash
git --version
```

✅ `git version 2.x.x`のような表示が出れば、すでにインストール済み！**Step 2へ進んでください**。

❌ `command not found`と表示された場合は、以下の手順でインストールしてください。

#### Windowsの場合:

1. ブラウザで以下のURLにアクセス:
   ```
   https://git-scm.com/download/win
   ```

2. 自動的にダウンロードが始まります（64-bit版）

3. ダウンロードされたファイル（`Git-2.xx.x-64-bit.exe`）をダブルクリック

4. インストーラーの設定（重要な部分のみ説明）:
   - 「Next」を数回クリック（デフォルトのままでOK）
   - 「Choosing the default editor used by Git」の画面:
     - 「Use Visual Studio Code as Git's default editor」を選択
   - それ以外はすべてデフォルトのまま「Next」
   - 「Install」をクリック

5. インストール完了後、**コマンドプロンプトを一度閉じて、新しく開く**

#### Macの場合:

Macには最初からGitが入っていることが多いですが、最新版を入れるのがおすすめです。

1. ターミナルを開く

2. 昨日インストールしたHomebrewを使ってインストール:
   ```bash
   brew install git
   ```

3. インストール完了を待つ（2〜3分）

**✅ 動作確認**:

**新しい**コマンドプロンプト/ターミナルを開いて:

```bash
git --version
```

✅ `git version 2.43.0`のようなバージョンが表示されればOK！

**🔍 コマンドの深掘り解説**:

- `git`: Gitコマンドの実行
- `--version`: Gitのバージョンを表示

### Step 2: プロジェクトをクローン（ダウンロード）

**🔰 初心者向け解説**:

「クローン（clone）」は、GitHubにあるコードを、自分のパソコンにダウンロードすることです。

> 💡 **例え話**:
> クローンは「図書館の本をコピーする」ようなものです。
>
> - **図書館の本**: GitHubに保存されているコード（元本）
> - **コピー**: あなたのパソコンにダウンロードされたコード
>
> 元の本は図書館に残ったまま。あなたはコピーを使って自由に書き込んだり、試したりできます。

**💻 実装方法**:

#### Step 2-1: 作業用フォルダを作る

まず、プロジェクトを保存する場所を決めましょう。

**Windowsの場合**:

1. コマンドプロンプトを開く

2. 以下のコマンドを1行ずつ実行:
   ```bash
   cd C:\Users\%USERNAME%\
   ```
   ```bash
   mkdir projects
   ```
   ```bash
   cd projects
   ```

これで、`C:\Users\[あなたの名前]\projects\`というフォルダが作られ、そこに移動しました。

**Macの場合**:

1. ターミナルを開く

2. 以下のコマンドを1行ずつ実行:
   ```bash
   cd ~
   ```
   ```bash
   mkdir -p projects
   ```
   ```bash
   cd projects
   ```

これで、`/Users/[あなたの名前]/projects/`というフォルダが作られ、そこに移動しました。

**🔍 コマンドの深掘り解説**:

- `cd`: Change Directory（フォルダを移動する）
- `mkdir`: Make Directory（フォルダを作る）
- `~`: ホームディレクトリ（あなたの個人フォルダ）
- `-p`: 親フォルダも同時に作る（すでにある場合はエラーにならない）

#### Step 2-2: プロジェクトをクローン

1. 現在地が`projects`フォルダにいることを確認:

   **macOS / Linux、または Windows で PowerShell / Git Bash を使っている場合**:
   ```bash
   pwd
   ```
   - `/Users/[あなたの名前]/projects` または `C:\Users\[あなたの名前]\projects` のような表示が出ればOK

   **Windows の「コマンドプロンプト（cmd）」を使っている場合**:
   ```bat
   cd
   ```
   - `C:\Users\[あなたの名前]\projects` のように `projects` で終わっていればOK

2. 以下のコマンドを実行して、プロジェクトをクローン:
   ```bash
   git clone https://github.com/kouiso/task-app.git
   ```

3. クローンが始まります（30秒〜1分程度）:
   ```
   Cloning into 'task-app'...
   remote: Enumerating objects: 1234, done.
   remote: Counting objects: 100% (1234/1234), done.
   ...
   Resolving deltas: 100% (789/789), done.
   ```
   このような表示が出て、最後に元の画面に戻ればOK！

4. クローンされたフォルダに移動:
   ```bash
   cd task-app
   ```

**✅ 動作確認**:

以下のコマンドで、フォルダの中身を確認:

**Windowsの場合**:
```bash
dir
```

**Macの場合**:
```bash
ls -la
```

たくさんのファイルやフォルダ名が表示されればOK！

次のようなものが見えるはずです:
- `package.json`
- `src/`
- `prisma/`
- `README.md`
など

**🔍 コマンドの深掘り解説**:

- `git clone [URL]`: 指定したURLからコードをダウンロード
- `dir` / `ls`: ファイル・フォルダの一覧を表示
- `-la`: 隠しファイルも含めて詳細表示（Macの場合）

### Step 4: 環境変数ファイルを作成する

**🔰 初心者向け解説**:

環境変数とは、「アプリの設定情報」を書いたファイルです。パスワードやデータベースの接続先など、「秘密の情報」をここに書きます。

> 💡 **例え話**:
> 環境変数ファイルは「金庫のダイヤル番号」を書いたメモのようなものです。
>
> - **金庫**: データベース
> - **ダイヤル番号**: パスワードや接続情報
> - **メモ**: .envファイル
>
> このメモは「自分だけが見る」ものなので、他の人に公開してはいけません。だから、Gitで管理する時も、このファイルは除外されています。
>
> 代わりに「.env.example」という「例」が用意されています。これをコピーして、自分用の設定を作ります。

**💻 実装方法**:

#### Step 4-1: .env.exampleを.envにコピー

VS Codeで作業します。

1. VS Codeの左側のファイル一覧で、`.env.example`を探す

2. `.env.example`を右クリック → 「コピー」

3. `task-app`フォルダ（一番上）を右クリック → 「貼り付け」

4. コピーされたファイル名が`.env.example のコピー`のようになっているので、これをクリックして名前を変更:
   ```
   .env
   ```
   （`.env`だけ。拡張子なし）

5. Enter キーを押して確定

**コマンドラインでやる場合**:

VS Codeの下部にある「ターミナル」タブ（またはメニューから「ターミナル」→「新しいターミナル」）で以下を実行:

**Windowsの場合**:
```bash
copy .env.example .env
```

**Macの場合**:
```bash
cp .env.example .env
```

#### Step 4-2: .envファイルの中身を確認

1. VS Codeで`.env`ファイルをクリックして開く

2. 以下のような内容が表示されます（一部抜粋）:
   ```bash
   # Docker Compose
   _DOCKER_COMPOSE_HOST_PORT_DB=5432
   _DOCKER_COMPOSE_HOST_PORT_BACKEND=3000

   # Database
   DATABASE_URL="postgresql://user:password@localhost:${_DOCKER_COMPOSE_HOST_PORT_DB}/taskapp?schema=public"

   # NextAuth
   NEXTAUTH_URL="http://localhost:3000"
   NEXTAUTH_SECRET="your-nextauth-secret-key-32-chars-minimum-please-change"
   ```

   > 📝 **補足**: 実際のファイルには他にも設定項目（開発者情報、JWT設定など）がありますが、今は表示されている部分だけ確認すればOKです。

**📝 今は変更しなくてOK！**

今は、このままで大丈夫です。それぞれの意味は後日学びます。

**🔍 内容の簡単な説明**:

- `_DOCKER_COMPOSE_HOST_PORT_*`: Docker で使用するポート番号の設定
- `DATABASE_URL`: データベースへの接続情報（住所とパスワード）
- `NEXTAUTH_URL`: アプリのURL
- `NEXTAUTH_SECRET`: 暗号化のための秘密の鍵

### Step 5: 環境を初期化する（task init）

**🔰 初心者向け解説**:

ここからが本番です！`task init`というコマンド1つで、以下の作業が**自動的に**実行されます：

1. 必要なパッケージ（部品）をインストール
2. Dockerコンテナ（データベース）を起動
3. データベースを作成
4. サンプルデータを投入

> 💡 **例え話**:
> `task init`は「新居の初日にやるべきこと」を全自動でやってくれるロボットのようなものです。
>
> - 家具を組み立てて（パッケージインストール）
> - 電気・ガス・水道を開通させて（Dockerコンテナ起動）
> - 冷蔵庫に食材を入れる（サンプルデータ投入）
>
> 本来なら数時間かかる作業を、コマンド1つで全部やってくれます。

**💻 実装方法**:

1. VS Codeのターミナルが開いていることを確認
   - 開いていない場合: メニューから「ターミナル」→「新しいターミナル」
   - VS Codeのターミナルは通常 PowerShell（Windows）または bash（Mac/Linux）が使われます

2. 現在地が`task-app`フォルダにいることを確認:

   **PowerShell / bash / Git Bash の場合**:
   ```bash
   pwd
   ```
   - `.../task-app` と表示されればOK

   **コマンドプロンプト（cmd）の場合**:
   ```bat
   cd
   ```
   - `...\task-app` で終わっていればOK

3. 以下のコマンドを実行:
   ```bash
   task init
   ```

4. **初回は10〜15分かかります**。以下のような処理が順番に実行されます:

   ```
   task: [clean] クリーンアップ中...
   task: [docker-rebuild] Dockerコンテナを再構築中...
   [+] Building ...
   ...
   task: [npmi] 依存パッケージをインストール中...
   added 1234 packages in 2m
   ...
   task: [db-apply] データベースをセットアップ中...
   Prisma schema loaded from prisma/schema.prisma
   ...
   task: [seed] シードデータを投入中...
   🌱 シードデータの投入を開始します
   👤 ユーザーを作成中...
   📁 プロジェクトを作成中...
   ✅ シードデータの投入が完了しました
   ```

5. 最後に以下のメッセージが表示されれば成功:
   ```
   task: 環境の初期化が完了しました！
   ```

**⏸️ もし途中でエラーが出たら**:

- **Docker Desktopが起動していますか？**
  - タスクバー（Windows）またはメニューバー（Mac）でDockerアイコン（クジラ）を確認
  - 起動していない場合は起動して、もう一度`task init`を実行

- **ポート5432が使われています、というエラー**:
  - 他のPostgreSQLが動いているかもしれません
  - 一度パソコンを再起動してから再実行

- **その他のエラー**:
  - エラーメッセージをよく読む
  - このドキュメントの「よくあるトラブル」セクションを確認

**✅ 動作確認**:

以下のコマンドで、Dockerコンテナが動いているか確認:

```bash
docker ps
```

以下のような表示が出ればOK:
```
CONTAINER ID   IMAGE         ...   STATUS         PORTS                    NAMES
abc123def456   postgres:16   ...   Up 2 minutes   0.0.0.0:5432->5432/tcp   task-app-db
```

**🔍 コマンドの深掘り解説**:

- `task init`: Taskfile.yamlに定義された「init」タスクを実行
- `docker ps`: 現在動いているDockerコンテナの一覧を表示
  - `ps`: Process Status（プロセスの状態）

### Step 6: 開発サーバーを起動する

**🔰 初心者向け解説**:

環境の準備ができたので、いよいよアプリを起動します！

> 💡 **例え話**:
> 開発サーバーを起動するのは「お店を開店する」ようなものです。
>
> - **準備完了**（Step 5まで）: 食材も揃った、厨房も整った
> - **開店**（これから）: 「OPEN」の札を掛けて、お客さんを迎える
>
> `localhost:3000`という住所で、あなただけがアクセスできるお店が開きます。

**💻 実装方法**:

1. VS Codeのターミナルで、以下のコマンドを実行:
   ```bash
   task up-backend
   ```

2. サーバーが起動します（30秒〜1分）:
   ```
   > task-app@1.0.0 dev
   > next dev

   ▲ Next.js 15.3.6
   - Local:        http://localhost:3000
   - Experiments:  ...

   ✓ Starting...
   ✓ Ready in 2.3s
   ```

3. `Ready in 2.3s`のようなメッセージが表示されれば成功！

**⚠️ 注意**: このターミナルは開いたままにしてください！閉じるとサーバーが停止します。

**🔍 コマンドの深掘り解説**:

- `task up-backend`: バックエンド（サーバー）を起動するタスク
- 内部的には`npm run dev`（Next.jsの開発モード）が実行されています

## 🧠 理解度チェック（復習）

### Q1: `git clone`コマンドは何をするコマンドですか？

<details>
<summary>答えを見る</summary>

**答え**: GitHubにあるコードを、自分のパソコンにダウンロード（クローン）するコマンドです。

**例え話**: 図書館の本をコピーするようなもので、元の本は図書館に残ったまま、自分用のコピーを手に入れます。

</details>

### Q2: `.env`ファイルは何のためのファイルですか？

<details>
<summary>答えを見る</summary>

**答え**: 環境変数（アプリの設定情報）を書くファイルです。データベースの接続情報やパスワードなど、秘密の情報を保存します。

**例え話**: 金庫のダイヤル番号を書いたメモのようなもので、自分だけが見る情報です。

</details>

### Q3: `task init`コマンドは何をしますか？

<details>
<summary>答えを見る</summary>

**答え**: 開発環境を初期化します。具体的には：
1. パッケージをインストール
2. Dockerコンテナを起動
3. データベースを作成
4. サンプルデータを投入

これらをすべて自動で実行してくれる便利なコマンドです。

</details>

### Q4: `localhost:3000`の`localhost`と`3000`はそれぞれ何を意味しますか？

<details>
<summary>答えを見る</summary>

- **3000**: ポート番号（アプリが動いている「出入り口」の番号）

**例え話**: localhostはマンション（建物）、3000は部屋番号のようなものです。

</details>

### Q5: 開発サーバーを停止するにはどうすればいいですか？

<details>
<summary>答えを見る</summary>

**答え**: 開発サーバーが動いているターミナルで、`Ctrl + C`（WindowsもMacも同じ）を押します。

これで、サーバーが停止します。再度起動したい場合は、また`task up-backend`を実行します。

</details>

## ❓ よくあるトラブルと解決方法

### トラブル 1: `task init`で「Docker is not running」というエラーが出る

**原因**: Docker Desktopが起動していない

**解決方法**:
1. タスクバー（Windows）またはメニューバー（Mac）でDockerアイコンを探す
2. アイコンがない、またはグレーアウトしている場合は、Docker Desktopを起動
3. クジラのアイコンが表示されて、正常に動いていることを確認
4. もう一度`task init`を実行

### トラブル 2: `localhost:3000`にアクセスしても「接続できません」と表示される

**原因**: 開発サーバーが起動していない、またはまだ起動中

**解決方法**:
1. ターミナルで「Ready in X.Xs」というメッセージが出ているか確認
2. 出ていない場合は、サーバーがまだ起動中か、エラーが出ている可能性があります
3. ターミナルのエラーメッセージをよく読む
4. それでも解決しない場合は、`Ctrl + C`でサーバーを停止し、`task up-backend`で再起動

### トラブル 3: ログインできない（「メールアドレスまたはパスワードが正しくありません」と表示される）

**原因**: 入力ミス、またはシードデータが投入されていない

**解決方法**:
1. メールアドレスとパスワードを再確認:
   - メールアドレス: `admin@example.com`（すべて小文字）
   - パスワード: `password123`（すべて小文字）
2. それでもダメな場合、シードデータが投入されていない可能性:
   - ターミナルで`Ctrl + C`でサーバー停止
   - `task seed`を実行（データベースをリセットしてサンプルデータを再投入）
   - `task up-backend`でサーバー再起動
   - もう一度ログイン

### トラブル 4: `code .`コマンドが動かない

**原因**: VS Codeのコマンドラインツールがインストールされていない

**解決方法**:
1. VS Codeを手動で起動
2. `Ctrl/Cmd + Shift + P`でコマンドパレットを開く
3. 「shell command」と入力
4. 「Shell Command: Install 'code' command in PATH」を選択
5. VS Codeを再起動
6. もう一度`code .`を試す

### トラブル 5: 画面が真っ白、または表示が崩れている

**原因**: ブラウザのキャッシュの問題、またはCSS/JSが正しく読み込まれていない

**解決方法**:
1. ブラウザで「スーパーリロード」を実行:
   - **Windows**: `Ctrl + Shift + R`
   - **Mac**: `Cmd + Shift + R`
2. それでもダメな場合、ブラウザのキャッシュをクリア:
   - Chromeの場合: 設定 → プライバシーとセキュリティ → 閲覧履歴データの削除
3. 開発サーバーを再起動（`Ctrl + C`→`task up-backend`）
