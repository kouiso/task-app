# 📖 用語集

このカリキュラムで登場する主要な技術用語を、日常の例えを交えて解説します。

---

## フロントエンド関連

### React
UIを構築するためのJavaScriptライブラリ。レゴブロックのように小さな「コンポーネント」を組み合わせて画面を作る。

### Next.js
Reactの上に構築されたフレームワーク。Reactが「エンジン」なら、Next.jsは「車」のようなもの。ルーティング、サーバーサイドレンダリング、APIルートなど、Webアプリに必要な機能が最初から揃っている。

### App Router
Next.js 13以降で導入されたルーティング方式。`app/`フォルダの構造がそのままURLになる。`app/dashboard/page.tsx`→`/dashboard`。

### コンポーネント
UIの再利用可能なパーツ。例えば「ボタン」「カード」「ナビゲーションバー」など。一度作れば何度でも使い回せる。

### Props
コンポーネントに渡すデータ。関数の引数のようなもの。親コンポーネントから子コンポーネントにデータを渡す手段。

### State（状態）
コンポーネント内で管理する変化するデータ。例：フォームの入力値、ダイアログの開閉状態。`useState`フックで管理する。

### Hook（フック）
React 16.8で導入された関数群。`useState`（状態管理）、`useEffect`（副作用処理）、`useMemo`（計算結果のキャッシュ）など。

### JSX / TSX
JavaScriptの中にHTMLのような構文を書ける拡張記法。TSXはTypeScript版。見た目はHTMLだが、実際にはJavaScriptのオブジェクトに変換される。

### Tailwind CSS
ユーティリティファーストのCSSフレームワーク。`className="text-red-500 font-bold"`のように、クラス名を組み合わせてスタイルを適用する。CSSファイルを別に書く必要がない。

### shadcn/ui
Radix UIをベースにした、コピー&ペースト方式のコンポーネントライブラリ。ボタン、ダイアログ、セレクトボックスなど、よく使うUIパーツが揃っている。

## バックエンド関連

### API（Application Programming Interface）
アプリケーション間でデータをやり取りする仕組み。レストランで言えば「メニュー（何ができるか）」と「注文方法（どうやってリクエストするか）」を定義したもの。

### tRPC
TypeScriptで型安全なAPIを構築するライブラリ。フロントエンドとバックエンドで型を共有するので、型の不一致によるバグが起きない。

### Prisma
データベースとTypeScriptコードの間を取り持つORM（Object-Relational Mapping）。SQLを直接書く代わりに、TypeScriptのコードでデータベース操作ができる。

### PostgreSQL
信頼性の高いオープンソースのリレーショナルデータベース。エクセルの表のように行と列でデータを管理するが、大量のデータを高速に処理できる。

### スキーマ（Schema）
データベースの設計図。どんなテーブルがあり、各テーブルにどんなカラムがあるかを定義する。Prismaでは`schema.prisma`ファイルに記述する。

### マイグレーション（Migration）
データベースのスキーマ変更を管理する仕組み。Gitがコードの変更履歴を管理するように、マイグレーションはデータベース構造の変更履歴を管理する。

### ミドルウェア（Middleware）
リクエストが処理される前に実行される処理。ビルの入り口の警備員のように、全てのリクエストをチェックして認証済みかどうか確認する。

### ルーター（Router）
APIのエンドポイントをまとめたもの。`projectRouter`ならプロジェクト関連のAPI、`taskRouter`ならタスク関連のAPIを定義する。

## 認証関連

### JWT（JSON Web Token）
ユーザーの認証情報を含む暗号化されたトークン。遊園地の「フリーパスチケット」のようなもの。一度ログインすると発行され、以降はこのトークンを見せるだけでアクセスできる。

### Cookie（クッキー）
ブラウザに保存される小さなデータ。JWTトークンを保存するために使用。有効期限を設定でき、HTTPリクエスト時に自動的にサーバーに送信される。

### bcryptjs
パスワードをハッシュ化（暗号化）するライブラリ。元のパスワードに戻せない一方向の暗号化を行い、データベースに安全に保存する。

### jose
JWTの生成・検証を行うライブラリ。Edge Runtime（サーバーレス環境）でも動作する軽量な実装。

### セッション（Session）
ユーザーのログイン状態を管理する仕組み。「ログイン中」「どのユーザーか」「権限は何か」といった情報を保持する。

## 開発ツール関連

### Git
ソースコードのバージョン管理システム。ファイルの変更履歴を記録し、いつでも過去の状態に戻せる。複数人での共同開発を可能にする。

### GitHub
Gitリポジトリのホスティングサービス。コードの保存、共有、レビュー、Issue管理などの機能を提供する。

### npm（Node Package Manager）
Node.jsのパッケージマネージャー。他の開発者が作ったライブラリをインストール・管理するツール。`package.json`で依存関係を管理する。

### TypeScript
JavaScriptに「型」を追加した言語。変数や関数の引数・返り値にどんなデータが入るかを明示できる。エディタの補完やエラー検出が強力になる。

### Biome
コードのリンティング（品質チェック）とフォーマット（整形）を行うツール。チームで統一されたコードスタイルを維持するために使う。

### Vitest
テストフレームワーク。書いたコードが期待通りに動くか自動で確認する。`expect(1 + 1).toBe(2)`のようにテストを書く。

### Vercel
Next.jsの開発元が提供するホスティングサービス。GitHubにプッシュするだけで自動デプロイされる。

### 環境変数（Environment Variables）
アプリケーションの設定値を外部から注入する仕組み。パスワードやAPIキーなど、コードに直接書きたくない値を`.env`ファイルで管理する。

### Turbopack
Next.js内蔵の高速バンドラー。開発サーバーの起動やコード変更の反映が高速。

## データベース関連

### テーブル（Table）
データベース内のデータを格納する構造。エクセルのシートに相当する。例：`users`テーブル、`tasks`テーブル。

### リレーション（Relation）
テーブル間の関連付け。例：タスクはプロジェクトに所属する（多対一）、プロジェクトには複数のメンバーがいる（多対多）。

### CRUD
Create（作成）、Read（読み取り）、Update（更新）、Delete（削除）の頭文字。データ操作の基本4操作。

### CUID
衝突しにくいユニークなIDを生成するアルゴリズム。データベースの主キーに使用。`cm1abc2def3ghi4jkl5mno6pqr`のような文字列。

## アーキテクチャ関連

### クライアントサイド / サーバーサイド
- **クライアントサイド**: ブラウザ上で実行される処理。`'use client'`と書いたコンポーネント。
- **サーバーサイド**: サーバー上で実行される処理。データベースアクセスやAPIルート。

### Server Component / Client Component
- **Server Component**: サーバーで描画されるコンポーネント。データベースに直接アクセスできるが、`useState`などのフックは使えない。
- **Client Component**: ブラウザで描画されるコンポーネント。インタラクティブなUI（クリック、フォーム入力）を扱う。

### Edge Runtime
サーバーレス環境の一種。世界中のエッジサーバーで実行され、ユーザーに近い場所で高速にレスポンスを返す。Next.jsのミドルウェアで使用。
