# 学びのロードマップ（30日カリキュラム全体像）

このカリキュラムは、30日かけて1本のタスク管理アプリを「動かす → 公開する → 機能を積む → 仕上げて公開する」順で育てます。各日は前の日の上に積み上がるので、飛ばさず順番に進むのがいちばんの近道です。

日々の目次は `00_カリキュラム目次.md` を、各日の詳しい手順は `dayXX_*.md` を見てください。このファイルは「どの日で何を身につけ、どの技術がどこで再び登場するか」を一枚で見渡すための地図です。

## 7つの局面

30日は7つの局面に分かれます。局面ごとに、終わったときに何ができるようになるかを示します。

| 局面 | 日 | テーマ | 終わったときにできること |
|------|-----|--------|------------------------|
| 1. 起動と公開 | Day 01–04 | 環境構築 → ダッシュボード → Git → Vercel公開 | 自分のアプリをインターネットのURLで開ける |
| 2. 認証とUI土台 | Day 05–08 | ログイン・登録フォーム → 認証バックエンド → 共通レイアウト | ログインが実際に動き、全ページ共通のサイドバーが整う |
| 3. プロジェクト管理 | Day 09–12 | 一覧 → 新規作成 → 編集・削除 → メンバー | プロジェクトを作り、直し、消し、人を追加できる |
| 4. タスク管理 | Day 13–16 | 一覧 → 作成 → 編集・削除 → ステータス・タイマー | タスクの基本操作一式と作業時間の計測ができる |
| 5. タスク周辺機能 | Day 17–20 | マイタスク → コメント → コメント編集 → 検索 | 協働と検索ができ、実用的なアプリに近づく |
| 6. 可視化と管理 | Day 21–25 | 統計 → グラフ → 週次レポート → ユーザー管理 → プロフィール | データを見える化し、管理者機能を持てる |
| 7. 仕上げと公開 | Day 26–30 | エラー対応 → 詳細画面 → 一括操作 → ユーザー詳細 → 本番公開 | バグに対処し、仕上げて完成版を公開できる |

## 技術のつながり（どこで習い、どこで再び使うか）

このカリキュラムは、同じ型を何度も使わせて定着させる作りです。ある型を最初に習う日と、それを応用する日を並べます。前の日でつまずくと後の日に響くので、再登場する型は特に大事にしてください。

| 技術の型 | 最初に習う日 | 応用・復習する日 |
|---------|-------------|------------------|
| react-hook-form + zod（フォーム） | Day 05 | Day 06・10・14・18・25・29 |
| ダイアログ（モーダルのフォーム） | Day 10 | Day 11・14・15・16 |
| tRPC useQuery（データ取得） | Day 08–09 | Day 13・17・20・21 ほか取得のある画面すべて |
| tRPC useMutation（作成・更新・削除） | Day 08・10 | Day 11・14・15・16・18・19・25 |
| tRPC手続きの自作（入力・処理・戻り値の3部品） | Day 09（getAll） | Day 10以降のサーバー側すべて |
| キャッシュの無効化（invalidate） | Day 10 | Day 11・15・18・19 |
| 権限チェック（本人・管理者の切り分け） | Day 07・12 | Day 19・24・29 |
| Recharts（グラフ） | Day 22 | Day 23 |

## 大きな2つの周回

このアプリの中心は「プロジェクト」と「タスク」です。この2つは、ほぼ同じ CRUD（作成・取得・更新・削除）の流れを別々にたどります。

- プロジェクトの周回: Day 09（取得） → Day 10（作成） → Day 11（更新・削除） → Day 12（メンバー）
- タスクの周回: Day 13（取得） → Day 14（作成） → Day 15（更新・削除） → Day 16（ステータス・作業時間記録）

プロジェクトの周回で身につけた型を、タスクの周回でもう一度なぞる構成です。2周目は同じ骨組みに中身を入れ替えるだけになるので、1周目のプロジェクト側を理解しておくと、2周目のタスク側が楽になります。

## 進め方のコツ

- 各日の冒頭にある「今日のゴール」を先に読み、完成形のイメージを持ってから手を動かす
- コードを写したあとは、その下の「なぜこう動くか」を必ず読む。写すだけでは次の日に応用が効かない
- つまずいたら、その日の「つまずきポイント」の表を見る。多くの詰まりはそこで解決する
