### Step 1: 自分のOS（オペレーティングシステム）を確認する

**🔰 初心者向け解説**:

OS（オペレーティングシステム）とは、パソコンの基本ソフトのことです。Windows、Mac、Linuxなどがあります。

> 💡 **例え話**:
> OSは「家の間取り」のようなものです。一戸建て（Windows）、マンション（Mac）、団地（Linux）では、電気のスイッチの位置や水道の蛇口の形が違いますよね？
> 同じように、OSによってソフトのインストール方法が少し違います。だから、まず自分の「家の間取り」を確認する必要があるんです。

**💻 確認方法**:

**Windowsの場合**:
1. キーボードの「Windows」キー（旗のマーク）を押す
2. 「設定」と入力して Enter
3. 「システム」→「バージョン情報」をクリック
4. 「エディション」の欄を確認

✅ **Windows 10 または Windows 11** であることを確認してください。

**Macの場合**:
1. 画面左上のリンゴマークをクリック
2. 「この Mac について」をクリック
3. macOS のバージョンを確認

✅ **macOS Catalina (10.15) 以降**であることを確認してください。

**📝 メモ**: 自分のOSとバージョンをメモ帳などに書いておきましょう。後で必要になります。

### Step 2: Node.jsをインストールする (Voltaを使用)

**🔰 初心者向け解説**:

Node.jsは、JavaScriptというプログラミング言語を動かすための「エンジン」です。
本カリキュラムでは、**Volta**というツールを使ってインストールします。Voltaを使うと、プロジェクトごとに最適なエンジンのバージョンを自動で切り替えてくれるため、トラブルが少なくなります。

> 💡 **例え話**:
> - **Node.js**: 車のエンジン（これがないと動かない）
> - **Volta**: 整備士さん（「この車にはこのエンジンがいいよ」と自動で選んでくれる）
>
> 整備士さん（Volta）を雇っておけば、エンジンの交換や管理を全部やってくれるので、あなたは運転（プログラミング）に集中できます。

**💻 インストール手順**:

#### Windowsの場合:

1. ブラウザで以下のURLにアクセス:
   ```
   https://github.com/volta-cli/volta/releases/download/v2.0.1/volta-2.0.1-windows-x86_64.msi
   ```
   （ダウンロードが始まらない場合は [Volta公式サイト](https://docs.volta.sh/guide/getting-started) の "Windows Installation" からインストーラーをダウンロードしてください）

2. ダウンロードされた `.msi` ファイルをダブルクリックして実行

3. インストーラーの指示に従って「Next」→「Install」→「Finish」をクリック

4. **重要**: インストール完了後、PowerShellまたはコマンドプロンプトを**一度閉じて、開き直して**ください。

5. 以下のコマンドを実行して、Voltaが入ったか確認:
   ```bash
   volta --version
   ```
   - バージョン番号が表示されればOK！

6. Node.js (バージョン24.11.1) をインストール:
   ```bash
   volta install node@24.11.1
   ```

#### Macの場合:

1. ターミナルを開く

2. 以下のコマンドをコピー＆ペーストして実行:
   ```bash
   curl https://get.volta.sh | bash
   ```

3. ターミナルを**一度閉じて、開き直して**ください。

4. 以下のコマンドを実行して、Voltaが入ったか確認:
   ```bash
   volta --version
   ```
   - バージョン番号が表示されればOK！

5. Node.js (バージョン24.11.1) をインストール:
   ```bash
   volta install node@24.11.1
   ```

**✅ 動作確認（Windows/Mac共通）**:

以下のコマンドを実行して、正しいバージョンが表示されるか確認しましょう。

```bash
node --version
```

- `v24.11.1` と表示されればOK！

**🔍 コマンドの深掘り解説**:

- `volta install ...`: 指定したツールのバージョンをインストールし、使えるようにします
- `node`: Node.jsを起動するコマンド

これで「エンジン」の準備は完了です！

### Step 3: Docker Desktopをインストールする

**🔰 初心者向け解説**:

Dockerは、「仮想の開発環境」を作るツールです。データベース（データを保存する場所）をこの中で動かします。

> 💡 **例え話**:
> Dockerは「レンタルキッチン」のようなものです。
>
> 自分の家のキッチンを改造するのは大変ですよね？でも、レンタルキッチンなら、使いたいときだけ借りて、不要になったら返せばいい。壊れても自分の家には影響がない。
>
> Dockerも同じで：
> - **必要な時だけ**データベースを起動できる
> - **何か壊れても**、元の環境には影響しない
> - **他の人と同じ環境**を簡単に作れる（みんな同じレンタルキッチンを使う感じ）

**💻 インストール手順**:

#### Windowsの場合:

1. ブラウザで以下のURLにアクセス:
   ```
   https://www.docker.com/products/docker-desktop/
   ```

2. 「Download for Windows」ボタンをクリック

3. ダウンロードされたファイル（`Docker Desktop Installer.exe`）をダブルクリック

4. インストーラーが起動します：
   - 「OK」をクリック（設定はデフォルトのまま）
   - インストールには5〜10分かかります（コーヒーを飲んで待ちましょう☕）
   - 「Close and restart」をクリックしてパソコンを再起動

5. 再起動後、Docker Desktopが自動的に起動します
   - 「Accept」（規約に同意）をクリック
   - 「Skip」（アンケートはスキップでOK）をクリック

**重要**: Windows の場合、WSL2（Windows Subsystem for Linux 2）のセットアップが必要な場合があります。Docker Desktopが自動的に案内してくれるので、指示に従ってください。

#### Macの場合:

1. ブラウザで以下のURLにアクセス:
   ```
   https://www.docker.com/products/docker-desktop/
   ```

2. 「Download for Mac」をクリック
   - **Apple Silicon (M1/M2/M3)**の場合: 「Apple Chip」を選択
   - **Intel Mac**の場合: 「Intel Chip」を選択

3. ダウンロードされたファイル（`Docker.dmg`）をダブルクリック

4. 表示されたウィンドウで、Dockerのアイコンを「Applications」フォルダにドラッグ&ドロップ

5. アプリケーションフォルダから「Docker」を起動
   - 「開く」をクリック（セキュリティ警告が出た場合）
   - パスワードを入力して「OK」
   - 「Accept」（規約に同意）をクリック

**✅ 動作確認**:

Docker Desktopが起動したら、画面上部のメニューバー（Windows）またはメニューバー（Mac）に「Docker」のアイコン（クジラのマーク）が表示されます。

1. コマンドプロンプト（Windows）またはターミナル（Mac）を開く
2. 以下のコマンドを入力:
   ```bash
   docker --version
   ```
3. `Docker version 24.x.x` のような表示が出ればOK！

**🔍 コマンドの深掘り解説**:

- `docker`: Dockerを操作するための基本コマンド
- `--version`: Dockerのバージョンを確認

Docker Desktopが動いていることを確認できました！

### Step 4: Taskをインストールする

**🔰 初心者向け解説**:

Taskは、「よく使うコマンド」を簡単に実行できるようにするツールです。

> 💡 **例え話**:
> Taskは「家電のリモコン」のようなものです。
>
> テレビを見るとき、本体のボタンを毎回押しに行くのは面倒ですよね？リモコンなら、ボタン1つで「電源ON」「チャンネル変更」「音量調整」ができます。
>
> Taskも同じで：
> - 本来は長いコマンドを打つ必要がある作業を
> - `task init`のような短いコマンド1つで実行できる
>
> 例えば、「データベースをリセットしてサンプルデータを入れる」という作業は、本来なら5〜6個のコマンドを順番に実行する必要がありますが、Taskを使えば`task seed`だけでOKです！

**💻 インストール手順**:

#### Windowsの場合（PowerShellを使用）:

1. スタートメニューから「PowerShell」を**管理者として実行**
   - スタートメニューで「PowerShell」と検索
   - 右クリック → 「管理者として実行」を選択

2. 以下のコマンドをコピーして貼り付け、Enter:
   ```powershell
   Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```
   - 確認を求められたら「Y」と入力してEnter

3. 以下のコマンドを実行:
   ```powershell
   irm get.scoop.sh | iex
   ```
   - これで「Scoop」（Windowsのパッケージマネージャー）がインストールされます

4. Scoopがインストールされたら、以下のコマンドでTaskをインストール:
   ```powershell
   scoop install task
   ```

#### Macの場合（Homebrewを使用）:

1. ターミナルを開く

2. Homebrewがインストールされているか確認:
   ```bash
   brew --version
   ```
   - バージョンが表示されればOK（次の手順3へ）
   - `command not found`と表示された場合は、以下のコマンドでHomebrewをインストール:
     ```bash
     /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
     ```
   - パスワードを求められたら入力（画面には表示されませんが、打っています）
   - Enter キーを押すよう指示が出たら押す
   - インストールには5〜10分かかります

3. Taskをインストール:
   ```bash
   brew install go-task
   ```

**✅ 動作確認**:

1. **新しい**コマンドプロンプト（Windows）またはターミナル（Mac）を開く
   - ※重要: 必ず新しいウィンドウを開いてください

2. 以下のコマンドを入力:
   ```bash
   task --version
   ```

3. `Task version: v3.x.x` のような表示が出ればOK！

**🔍 コマンドの深掘り解説**:

- `task`: Taskコマンドの実行
- `--version`: バージョン確認

### Step 5: VS Codeをインストールする

**🔰 初心者向け解説**:

VS Code（Visual Studio Code）は、プログラムを書くための「エディタ」です。

> 💡 **例え話**:
> VS Codeは「高機能なワープロ」のようなものです。
>
> - **メモ帳**だと、ただ文字を書くだけ
> - **Word**だと、文字の装飾や校正機能がある
> - **VS Code**だと、コードの色分け、自動補完、エラー検出などができる
>
> コードを書くための最高の道具です。無料で使えて、世界中の開発者が使っています！

**💻 インストール手順**:

#### Windows/Mac共通:

1. ブラウザで以下のURLにアクセス:
   ```
   https://code.visualstudio.com/
   ```

2. 「Download for Windows」または「Download for Mac」をクリック

3. ダウンロードされたファイルを開く:
   - **Windows**: `VSCodeUserSetup-x64-x.xx.x.exe`
   - **Mac**: `VSCode-darwin-universal.zip`（解凍されて`.app`ファイルが出てきます）

4. インストーラーの指示に従う:
   - **Windows**:
     - 「同意する」にチェック → 「次へ」
     - インストール先はそのまま → 「次へ」
     - 「デスクトップ上にアイコンを作成する」にチェック
     - **重要**: 「PATHへの追加」にチェック → 「次へ」
     - 「インストール」をクリック

   - **Mac**:
     - `.app`ファイルを「アプリケーション」フォルダにドラッグ&ドロップ

5. インストール完了後、VS Codeを起動
   - **Windows**: デスクトップのアイコンをダブルクリック
   - **Mac**: アプリケーションフォルダから起動

**✅ 動作確認**:

VS Codeが起動し、画面左側にファイルツリー（まだ空）、中央にエディタ画面が表示されればOK！

### Step 6: VS Codeの拡張機能をインストールする

**🔰 初心者向け解説**:

拡張機能は、VS Codeに追加できる「便利ツール」です。スマホのアプリのようなものです。

> 💡 **例え話**:
> 拡張機能は「車のオプション装備」のようなものです。
>
> - 車は買った時点で走れますが、カーナビやバックカメラをつけるともっと便利ですよね？
> - VS Codeも、そのままでもコードは書けますが、拡張機能を入れるともっと快適になります。
>
> 例えば：
> - **日本語化**: メニューが日本語になる
> - **Prettier**: コードを自動で整形してくれる（スペースや改行を揃える）
> - **ESLint**: コードのミスを自動で見つけてくれる

**💻 実装方法**:

#### Step 6-1: 日本語化

1. VS Codeを開く

2. 画面左側のサイドバーにある「拡張機能」アイコン（四角が4つのマーク）をクリック
   - またはキーボードで `Ctrl + Shift + X`（Windows）/ `Cmd + Shift + X`（Mac）

3. 検索ボックスに以下を入力:
   ```
   Japanese Language Pack
   ```

4. 「Japanese Language Pack for Visual Studio Code」を探して「Install」（インストール）をクリック

5. インストール完了後、右下に「再起動が必要」というメッセージが出ます
   - 「Restart」をクリックしてVS Codeを再起動

6. 再起動後、メニューが日本語になっていればOK！

#### Step 6-2: 必須の拡張機能をインストール

同じ手順で、以下の拡張機能もインストールしましょう:

1. **ESLint**
   - 検索ボックスに `ESLint` と入力
   - 「ESLint」（Microsoft発行）をインストール
   - 役割: JavaScriptのコードをチェックして、ミスを教えてくれる

2. **Prettier - Code formatter**
   - 検索ボックスに `Prettier` と入力
   - 「Prettier - Code formatter」をインストール
   - 役割: コードを自動できれいに整形してくれる

3. **Prisma**
   - 検索ボックスに `Prisma` と入力
   - 「Prisma」をインストール
   - 役割: データベースのスキーマファイル（設計図）を見やすくする

4. **Docker**
   - 検索ボックスに `Docker` と入力
   - 「Docker」（Microsoft発行）をインストール
   - 役割: Dockerコンテナの管理を簡単にする

**🔍 拡張機能の深掘り解説**:

- **ESLint**: コードのスペルミスや文法ミスを、波線で教えてくれます（Wordの赤線みたいなもの）
- **Prettier**: ファイルを保存したときに、自動でコードをきれいに整えてくれます
- **Prisma**: データベースの設計図ファイルに色をつけて見やすくしてくれます
- **Docker**: Dockerコンテナの起動・停止をVS Code内でできるようになります


✅ `Docker version 24.x.x`のような表示が出る

Docker Desktopを起動して、クジラのアイコンが表示されていることを確認。

#### 確認 3: Task

```bash
task --version
```

✅ `Task version: v3.x.x`のような表示が出る

#### 確認 4: VS Code

2. メニューが日本語になっている

## ❓ よくあるトラブルと解決方法

### トラブル 1: Node.jsのインストーラーが開かない

**原因**: ダウンロードが完全に終わっていない、またはファイルが破損している

**解決方法**:
1. ブラウザのダウンロード履歴を確認し、ダウンロードが100%完了しているか確認
2. 完了していない場合は、再度ダウンロード
3. それでも開かない場合は、ダウンロードしたファイルを削除して、もう一度ダウンロード

### トラブル 2: Docker Desktopが起動しない

**原因**: システム要件を満たしていない、または仮想化が有効になっていない

**解決方法**:

**Windowsの場合**:
1. BIOSで仮想化（Virtualization）が有効になっているか確認
2. WSL2がインストールされているか確認
3. Dockerの公式ドキュメントのトラブルシューティングを参照: https://docs.docker.com/desktop/troubleshoot/overview/

**Macの場合**:
1. macOSのバージョンが要件を満たしているか確認（Catalina 10.15以降）
2. システム環境設定でDockerの実行を許可

### トラブル 3: `task --version`で「command not found」と表示される

**原因**: PATHが通っていない、またはターミナルを再起動していない

**解決方法**:
1. ターミナル/コマンドプロンプトを完全に閉じる
2. 新しいウィンドウを開いて再度実行
3. それでもダメな場合は、パソコン自体を再起動
4. 再起動後も解決しない場合は、Taskの再インストール

### トラブル 4: VS Codeが日本語にならない

**原因**: 拡張機能のインストール後に再起動していない

**解決方法**:
1. VS Codeを完全に終了（右上の×ボタンで閉じる）
2. もう一度VS Codeを起動
3. それでも日本語にならない場合:
   - `Ctrl/Cmd + Shift + P`でコマンドパレットを開く
   - 「Configure Display Language」と入力
   - 「ja」（日本語）を選択
   - VS Codeを再起動
