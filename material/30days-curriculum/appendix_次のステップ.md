## レベル1: Task-Appを拡張する

まずはこのカリキュラムで作ったTask-Appに新機能を追加してみましょう。コードベースを熟知しているので、最も取り組みやすいステップです。

### おすすめの拡張機能

| 機能 | 難易度 | 学べること |
|------|--------|------------|
| 通知機能 | ★★☆ | リアルタイム通信、状態管理 |
| ファイル添付 | ★★☆ | ファイルアップロード、クラウドストレージ連携 |
| カンバンボード表示 | ★★★ | ドラッグ&ドロップ、複雑なUI状態管理 |
| メール通知 | ★★☆ | 外部APIの連携、非同期処理 |
| ダークモード | ★☆☆ | テーマシステム、CSS変数 |
| 多言語対応（i18n） | ★★☆ | 国際化ライブラリ、翻訳管理 |
| ガントチャート | ★★★ | 日付計算、SVG/Canvas描画 |
| アクティビティログ | ★★☆ | イベント駆動設計、ログ設計 |

### 取り組み方
1. まず仕様を紙やメモに書き出す
2. 必要なデータベース変更を考える（Prismaスキーマ）
3. APIを設計する（tRPCルーター）
4. UIを実装する（Reactコンポーネント）
5. テストを書く

## レベル2: 新しいプロジェクトに挑戦

Task-Appで学んだ技術スタックを使って、ゼロから新しいアプリを作ってみましょう。

### プロジェクトアイデア

| プロジェクト | 概要 | 新しく学べること |
|-------------|------|------------------|
| ブログプラットフォーム | 記事のCRUD、マークダウン対応 | リッチテキストエディタ、SEO対策 |
| 家計簿アプリ | 収支管理、グラフ表示 | データ集計、チャート応用 |
| レシピ管理アプリ | レシピのCRUD、検索、カテゴリ | 画像管理、全文検索 |
| チャットアプリ | リアルタイムメッセージング | WebSocket、リアルタイム通信 |
| ECサイト | 商品管理、カート、決済 | 決済API連携、トランザクション管理 |

### ゼロから始めるときの手順
```bash
# 1. Next.jsプロジェクトの作成
npx create-next-app@latest my-new-app --typescript --tailwind --app

# 2. 必要なパッケージの追加
npm install @trpc/server @trpc/client @trpc/react-query
npm install prisma @prisma/client
npm install zod

# 3. Prismaの初期化
npx prisma init

# 4. 開発開始！
npm run dev
```

## レベル3: 技術を深掘りする

特定の技術領域をより深く学ぶことで、専門性を高めましょう。

### フロントエンド特化
- **パフォーマンス最適化**: React.memo、useMemo、useCallback、コード分割
- **アクセシビリティ（a11y）**: WAI-ARIA、キーボードナビゲーション、スクリーンリーダー対応
- **アニメーション**: Framer Motion、CSS Animations
- **状態管理の深化**: Zustand、Jotai、React Query（TanStack Query）の高度な使い方

### バックエンド特化
- **データベース設計**: 正規化、インデックス設計、クエリ最適化
- **認証・認可の深化**: OAuth 2.0、RBAC（ロールベースアクセス制御）
- **API設計**: REST vs GraphQL vs tRPC の比較、ページネーション、レートリミット
- **セキュリティ**: OWASP Top 10、CSRF対策、XSS対策、SQLインジェクション対策

### インフラ・DevOps
- **CI/CD**: GitHub Actions、自動テスト、自動デプロイ
- **コンテナ**: Docker Compose、マルチステージビルド
- **モニタリング**: エラートラッキング（Sentry）、ログ管理
- **スケーリング**: キャッシュ（Redis）、CDN、ロードバランシング

## レベル4: チーム開発を経験する

一人での開発スキルが身についたら、チームでの開発に挑戦しましょう。

### チーム開発で重要なスキル
- **コードレビュー**: Pull Requestのレビュー方法、建設的なフィードバック
- **ブランチ戦略**: Git Flow、GitHub Flow の理解と実践
- **Issue管理**: 要件の分割、見積もり、優先度付け
- **ドキュメント**: READMEの書き方、APIドキュメント、アーキテクチャ文書

### おすすめの活動
1. **OSS（オープンソースソフトウェア）に貢献する**
   - `good first issue`ラベルのIssueから始める
   - ドキュメントの修正やtypoの修正も立派な貢献

2. **ハッカソンに参加する**
   - 短期間でチーム開発を経験できる
   - 異なるスキルセットのメンバーとの協働を学ぶ

3. **勉強会に参加する**
   - connpassやmeetupでイベントを探す
   - LT（ライトニングトーク）で発表すると学びが深まる
